# Module 5: Multi-Format Document Support

## Context
The app currently only supports `.txt` and `.md` files via simple UTF-8 decode. Module 5 adds PDF, DOCX, and HTML support using the **docling** library (by IBM), which converts any document format to markdown text. This slots cleanly into the existing pipeline â€” parsing happens before chunking, so everything downstream (chunking, embedding, metadata extraction, record manager) stays unchanged.

Cascade deletes are already in place (`ON DELETE CASCADE` on `document_chunks.document_id` FK in migration 006).

## Task 2: Create Parsing Service

**New file:** `backend/services/parsing_service.py`

Single function: `extract_text(file_bytes: bytes, mime_type: str, filename: str) -> str`

Routing logic by MIME type:
| MIME Type | Strategy |
|-----------|----------|
| `text/plain` | `file_bytes.decode("utf-8")` â€” no docling needed |
| `text/markdown` | `converter.convert_string(decoded, format=InputFormat.MD).document.export_to_markdown()` |
| `application/pdf` | Write to temp file â†’ `converter.convert(path).document.export_to_markdown()` |
| `application/vnd.openxmlformats-officedocument.wordprocessingml.document` | Write to temp file â†’ same pattern |
| `text/html` | `converter.convert_string(decoded, format=InputFormat.HTML).document.export_to_markdown()` |

Implementation notes:
- Lazy-init a module-level `DocumentConverter` (reusable, heavy to construct)
- Use `tempfile.NamedTemporaryFile(suffix=ext, delete=False)` + `try/finally` cleanup for file-based formats
- Raise `ValueError` on unsupported type or empty extraction result
