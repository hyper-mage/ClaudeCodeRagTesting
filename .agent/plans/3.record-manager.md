# Module 3: Record Manager

## Context

Modules 1 & 2 are complete. The ingestion pipeline works (upload → chunk → embed → store → retrieve), but it's **naive** — it has no memory. Uploading the same file twice creates fully duplicate records and chunks, wasting embedding API calls and polluting search results. Re-uploading a modified file creates all-new chunks instead of updating only what changed.

Module 3 adds a **Record Manager**: content hashing at file and chunk levels, duplicate detection before processing, and incremental updates that diff chunk hashes to skip unchanged content.

## Complexity: ⚠️ Medium

5 tasks in 3 waves. Core logic is a single new service with well-defined inputs/outputs. Migration is additive. Frontend changes are minimal.

## Wave 1: Database Migration

### Task 1: Add Content Hash Columns

**Goal:** Add `content_hash` to `documents` and `document_chunks` tables.

**File:** `supabase/migrations/011_add_content_hashes.sql` *(new)*

```sql
-- File-level content hash (SHA-256 of raw file bytes)
ALTER TABLE documents ADD COLUMN content_hash TEXT;

-- Chunk-level content hash (SHA-256 of chunk text)
ALTER TABLE document_chunks ADD COLUMN content_hash TEXT;

-- Unique index: same user can't have two documents with identical content
-- Partial index (WHERE NOT NULL) so existing rows without hashes are unaffected
CREATE UNIQUE INDEX idx_documents_user_content_hash
  ON documents (user_id, content_hash)
  WHERE content_hash IS NOT NULL;

-- Index for fast chunk hash lookups during incremental updates
CREATE INDEX idx_chunks_content_hash
  ON document_chunks (document_id, content_hash);
```

**Design:** Columns are nullable so existing Module 2 documents continue working without a backfill. No `UNIQUE` on chunk hashes — same text can legitimately appear across documents.

**Validation:**
1. Run migration in Supabase SQL Editor
2. `SELECT content_hash FROM documents LIMIT 1;` — returns NULL for existing rows
3. Insert two documents with same `(user_id, content_hash)` — second fails (unique constraint)

## Wave 2: Backend Logic

### Task 2: Record Manager Service

**Goal:** New service providing content hashing and chunk diffing logic.

**File:** `backend/services/record_manager.py` *(new)*

**Functions:**

```python
import hashlib
from database import get_supabase

def hash_content(content: bytes) -> str:
    """SHA-256 hash of raw file bytes for file-level duplicate detection."""
    return hashlib.sha256(content).hexdigest()

def hash_chunk(text: str) -> str:
    """SHA-256 hash of chunk text for chunk-level change detection."""
    return hashlib.sha256(text.encode("utf-8")).hexdigest()

def check_duplicate(user_id: str, content_hash: str) -> dict | None:
    """Check if user already has a completed document with this content hash.
    Returns existing document record or None."""
    db = get_supabase()
    result = (
        db.table("documents")
        .select("*")
        .eq("user_id", user_id)
        .eq("content_hash", content_hash)
        .eq("status", "completed")
        .maybe_single()
        .execute()
    )
    return result.data

def find_previous_version(user_id: str, filename: str) -> dict | None:
    """Find a completed document with the same filename (for incremental update)."""
    db = get_supabase()
    result = (
        db.table("documents")
        .select("*")
        .eq("user_id", user_id)
        .eq("filename", filename)
        .eq("status", "completed")
        .maybe_single()
        .execute()
    )
    return result.data

def get_existing_chunk_hashes(document_id: str) -> dict[str, str]:
    """Get all chunk hashes for a document. Returns {content_hash: chunk_id}."""
    db = get_supabase()
    result = (
        db.table("document_chunks")
        .select("id, content_hash")
        .eq("document_id", document_id)
        .execute()
    )
    return {row["content_hash"]: row["id"] for row in result.data if row["content_hash"]}

def diff_chunks(
    old_hashes: dict[str, str],   # {content_hash: chunk_id}
    new_hashes: list[str],         # content_hashes of new chunks in order
) -> tuple[list[int], list[str]]:
    """Compare old and new chunk hashes.
    Returns (new_chunk_indices, stale_chunk_ids)."""
    old_set = set(old_hashes.keys())
    new_set = set(new_hashes)
    new_indices = [i for i, h in enumerate(new_hashes) if h not in old_set]
    stale_ids = [cid for h, cid in old_hashes.items() if h not in new_set]
    return new_indices, stale_ids
```

**Key decisions:**
- `check_duplicate` only matches `status: completed` — failed uploads can be retried
- `diff_chunks` is a pure function (easy to test)
- Only `hashlib` (stdlib) — no new dependencies

**Validation:**
1. `hash_content(b"hello")` returns consistent SHA-256 hex string
2. `diff_chunks({"abc": "id1", "def": "id2"}, ["abc", "xyz"])` → `([1], ["id2"])`

### Task 3: Update Upload Router + Ingestion Service

**Goal:** Wire record manager into upload flow with three paths:
- **Path A** — Exact duplicate (same content hash) → return existing document
- **Path B** — Incremental update (same filename, different content) → diff and process only changes
- **Path C** — Fresh upload → full processing (unchanged)

#### Upload Router Changes (`documents.py`)

Updated `upload_document` flow:

```python
from services.record_manager import hash_content, check_duplicate, find_previous_version

@router.post("/upload")
async def upload_document(file, user_id):
    content = await file.read()
    content_hash = hash_content(content)

    # Path A: Exact duplicate
    existing = check_duplicate(user_id, content_hash)
    if existing:
        return {**existing, "duplicate": True, "message": "This file has already been uploaded"}

    # Path B: Check for same filename (incremental update candidate)
    previous = find_previous_version(user_id, filename)

    # Upload to storage + create document record (add content_hash field)
    doc = db.table("documents").insert({
        ...existing_fields,
        "content_hash": content_hash,  # NEW
    }).execute()

    # Process
    if previous and previous["content_hash"]:  # Only incremental if old doc has hash
        process_document_incremental(doc_id, user_id, previous["id"])
    else:
        process_document(doc_id, user_id)
```

HTTP status stays 200 for duplicates — it's not an error, the file is ready to use.

**2. New `process_document_incremental()`:**

```python
def process_document_incremental(new_doc_id: str, user_id: str, old_doc_id: str) -> None:
    """Incremental processing: diff chunks, embed only new ones, reuse unchanged."""
    # 1. Download + chunk new file
    # 2. Hash each new chunk
    # 3. Get old chunk hashes via get_existing_chunk_hashes(old_doc_id)
    # 4. diff_chunks() → new_indices, stale_ids
    # 5. Delete stale chunks from old document
    # 6. Re-parent surviving chunks (update document_id + chunk_index to new doc)
    # 7. Embed + insert only new chunks
    # 8. Delete old document record + storage file
    # 9. Update new document status to completed
```

Re-parenting means surviving chunks get `document_id` updated from old → new doc, preserving their embeddings. The old document is then deleted (no orphan chunks since they've been moved).

3. Upload modified version of same filename → only changed/new chunks get embedded

5. Upload same content with different filename → duplicate detected (by hash, not name)
