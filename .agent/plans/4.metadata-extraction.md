# Module 4: Metadata Extraction

## Context

Modules 1-2 established the full RAG pipeline: upload files, chunk text, embed chunks, store in pgvector, retrieve via tool calling. Currently the only metadata stored per chunk is `{"filename": "...", "chunk_index": N}` — purely structural, no semantic content.

Module 4 adds **LLM-powered structured metadata extraction** so the system can classify documents and filter retrieval by document type, topic, etc. This teaches: Pydantic structured extraction, JSONB schema design, and metadata-enhanced retrieval.

**Key design decision:** Doc-level extraction (one LLM call per document from first ~4000 chars), stored on `documents` table and propagated into each chunk's `metadata` JSONB. This is cheap, consistent, and enables filtering at the chunk level during vector search.

## Complexity: ⚠️ Medium
7 tasks across 4 waves. Core challenge is wiring metadata through 3 layers (DB → retrieval service → tool definition).

## Wave 2: Extraction Service + DB Migration (parallel)

### Task 2: Metadata Extraction Service

**Goal:** New service that calls the LLM to extract structured metadata using Pydantic + Chat Completions.

**Files:** `backend/services/metadata_service.py` (new)

**Implementation:**
- `extract_metadata(text: str) -> DocumentMetadata` — uses `get_llm_client()` from `llm_service.py` (line 11) for a non-streaming `client.chat.completions.create()` call
  - Truncates input to first 4000 chars
  - System prompt: "Extract structured metadata from the document. Respond with valid JSON matching the schema."
  - Uses `response_format` with `json_schema` for structured output (modern approach)
  - Parses response with `DocumentMetadata.model_validate_json()`
- `extract_metadata_safe(text: str) -> DocumentMetadata` — wraps `extract_metadata` with try/except, returns sensible defaults on failure so ingestion is never blocked

**Validation:** Call with sample text (e.g., a Python tutorial), verify returned `DocumentMetadata` has reasonable `document_type="tutorial"`, topic, keywords. Call `extract_metadata_safe("")` to confirm graceful fallback.

### Task 3: Database Migration

**Goal:** Add metadata column to `documents` table, GIN index on `document_chunks.metadata`, and update the `match_document_chunks` RPC to accept an optional metadata filter.

**Files:** `supabase/migrations/011_add_document_metadata.sql` (new)

**SQL:**
```sql
-- 1. Add metadata JSONB column to documents table
ALTER TABLE documents ADD COLUMN IF NOT EXISTS metadata JSONB DEFAULT '{}';

-- 2. GIN index on document_chunks.metadata for @> containment queries
CREATE INDEX IF NOT EXISTS idx_document_chunks_metadata
ON document_chunks USING GIN (metadata);

-- 3. Update RPC with optional filter_metadata parameter
CREATE OR REPLACE FUNCTION match_document_chunks(
  query_embedding VECTOR(2048),
  match_count INT DEFAULT 5,
  filter_user_id UUID DEFAULT NULL,
  filter_metadata JSONB DEFAULT NULL
)
RETURNS TABLE (
  id UUID, document_id UUID, content TEXT, chunk_index INT, similarity FLOAT
)
LANGUAGE plpgsql AS $$
BEGIN
  RETURN QUERY
  SELECT dc.id, dc.document_id, dc.content, dc.chunk_index,
    1 - (dc.embedding <=> query_embedding) AS similarity
  FROM document_chunks dc
  WHERE dc.user_id = filter_user_id
    AND (filter_metadata IS NULL OR dc.metadata @> filter_metadata)
  ORDER BY dc.embedding <=> query_embedding
  LIMIT match_count;
END;
$$;
```

**Validation:** Run migration, verify column exists, test RPC with `filter_metadata = NULL` (backward compat) and with a sample filter.

## Wave 3: Pipeline Integration (parallel)

### Task 4: Integrate Extraction into Ingestion

**Goal:** Call `extract_metadata_safe()` during `process_document()` and store results at both document and chunk levels.

**Files:** `backend/services/ingestion_service.py` (lines 50-118)

**Changes to `process_document()`:**
1. After downloading file text (line 72) and before chunking (line 78), call `extract_metadata_safe(text)`
2. Store result on documents table: `db.table("documents").update({"metadata": metadata_dict}).eq("id", doc_id).execute()`
3. Merge metadata into chunk rows (line 99): change from `{"filename": ..., "chunk_index": i}` to `{"filename": ..., "chunk_index": i, **metadata_dict}`

**Pipeline order:** download → extract metadata → store doc metadata → chunk → embed → insert chunks (with merged metadata) → mark completed

**Validation:** Upload a `.txt` file, check `documents.metadata` has extracted JSON, check `document_chunks.metadata` has merged fields (filename + chunk_index + extracted metadata). Upload empty-ish file to confirm graceful fallback.
