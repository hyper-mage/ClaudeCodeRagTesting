# Milestone v1.2: User Options & BYOK

**Status:** ✅ SHIPPED 2026-07-11
**Phases:** 9-15 + backlog 999.1/999.2
**Total Plans:** 43

## Overview

Users run LLMs of their choice from their own OpenRouter keys with near-zero-friction onboarding (one-click OAuth PKCE, no manual key paste), while a gated owner-key demo fallback preserves the public demo. Secret custody was the milestone's declared defining risk: keys are encrypted at rest (MultiFernet), resolved per-request with zero cross-user bleed, and provably absent from LangSmith traces, Sentry events, logs, and SSE payloads — both mandatory live SEC-01 gates passed on prod 2026-07-11.

## Milestone Summary

**Decimal/Inserted Phases:**

- Phase 999.1: Chat Empty-State UX (promoted from backlog mid-milestone)

**Key Decisions:**

- OAuth-only BYOK posture — no manual key paste (halves failure + security surface)
- MultiFernet from day one — rotation without migration (new-key-first `KEY_ENCRYPTION_SECRET` list)
- Lazy TTL model cache, no in-process scheduler (Fly free-tier suspend kills timers)
- Security front-loaded as release blockers; demo-fallback flag deliberately LAST, hard-gated on the 999.2 live trip test
- LangSmith gate at the run layer + runtime DB master toggle (after the client-wrap-only gate leaked BYOK turns via the endpoint `@traceable`)
- Per-request uncached key/model resolution — fresh client per call
- `default_model` plain TEXT, no FK to `model_cache` (deprecated-pin slug must persist for the at-send fallback notice)
