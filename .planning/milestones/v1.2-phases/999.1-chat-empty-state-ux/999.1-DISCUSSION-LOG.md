## Empty-state behavior (core fork)

| Option | Description | Selected |
|--------|-------------|----------|
| Auto-create thread on send | First message silently creates a thread then sends; composer always enabled (ChatGPT/Claude style) | ✓ |

**Notes:** Zero-friction is the intended UX; the message must "just work" on first send.
