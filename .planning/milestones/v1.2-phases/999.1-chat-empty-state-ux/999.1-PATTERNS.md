### `frontend/src/hooks/useChat.ts` (hook, streaming + optimistic state)

**Analog:** `sendMessage` itself ŌĆö an in-place refactor. The optimistic machinery, SSE loop, and error path are already correct; only the line-55 guard and the `loadMessages` clobber need changing.

**Root-cause guard to refactor** (lines 54-55) ŌĆö `threadId` is captured from the stale closure; an explicit param must win:
```typescript
const sendMessage = useCallback(async (content: string, opts?: { retry?: boolean }) => {
  if (!threadId || isStreaming) return            // ŌåÉ null threadId silently no-ops (the bug)
```
**Target signature** (add `threadId` to `opts`; mirror the existing `retry` opt; deps stay `[threadId, isStreaming, showToast]`):
```typescript
const sendMessage = useCallback(async (
  content: string,
  opts?: { retry?: boolean; threadId?: string }
) => {
  const effectiveThreadId = opts?.threadId ?? threadId   // explicit wins over closure
  if (!effectiveThreadId || isStreaming) return
  // ...use effectiveThreadId for every `/api/threads/${...}/messages` URL (lines 82-84)...
}, [threadId, isStreaming, showToast])
```

**Existing `opts` convention to mirror** (the `retry` flag is the precedent for adding `threadId`):
```typescript
const isRetry = opts?.retry === true
```

**Optimistic-bubble machinery to keep unchanged** (lines 61-73) ŌĆö this is the behavior the guard currently blocks; it must run synchronously before any `loadMessages` so the empty-state vanishes on first send:
```typescript

const assistantId = crypto.randomUUID()
```

**SSE URL construction to update to `effectiveThreadId`** (lines 82-84):
```typescript
const url = isRetry
  ? `/api/threads/${threadId}/messages?retry=true`

```

**`loadMessages` clobber guard (Pitfall 2 option 2)** ŌĆö current loader (lines 38-52) unconditionally replaces `messages`; guard it so it does not wipe a fresh optimistic/streaming thread:
```typescript
const loadMessages = useCallback(async () => {
  if (!threadId) {
    setMessages([])
    return

  // candidate guard: `if (isStreaming) return` ŌĆö do not clobber an in-flight optimistic send
  try {

    setMessages(data.messages.map(...))
  } catch { /* preserve silent-on-error behavior */ }

```

**Error / Sentry / toast pattern to mirror for the new create-failure path (Pitfall 5)** (lines 208-237) ŌĆö reuse this exact shape; do NOT add new error copy, reuse the existing toast surface:

**Source:** `frontend/src/hooks/useChat.ts:208-237` (catch block) + `frontend/src/contexts/ToastContext.tsx:42-49` (`showToast(message, variant)`)
