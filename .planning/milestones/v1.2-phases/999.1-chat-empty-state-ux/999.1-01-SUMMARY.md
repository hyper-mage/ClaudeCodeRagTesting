- Installed and wired the standard Vite/React 19 component-test stack: `vitest` + `@testing-library/react` + `@testing-library/jest-dom` + `@testing-library/user-event` + `jsdom`, behind `npm run test` (run mode, no watch).

## Decisions Made
- **Auth-in-tests via vi.mock factory, not by exporting the production AuthContext.** An earlier attempt exported `AuthContext` from `AuthContext.tsx` so the helper could mount a value, but that added a NEW `react-refresh/only-export-components` error to a production file (the file already had 1 pre-existing violation on `useAuth`). Reverted; auth is now supplied via `makeAuthMock()` reading a shared `authMockState`, leaving the production file byte-identical to its prior state.
- **Disabled react-refresh for test files in ESLint.** Test utility modules legitimately co-export a provider component, render helpers, and mock factories; they are never HMR-loaded, so the rule does not apply. Scoped to `**/*.test.{ts,tsx}` and `src/test/**`.
- **`sendMessage` deps unchanged** (`[threadId, isStreaming, showToast]`): the explicit id is read from the call argument, not a new dependency — mirrors the existing `opts?.retry` precedent exactly.
