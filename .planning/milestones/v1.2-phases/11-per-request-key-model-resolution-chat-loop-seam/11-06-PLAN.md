<objective>
Layer a runtime-flippable GLOBAL LangSmith master toggle ABOVE the Phase 11-05
per-request BYOK gate. The owner wants to turn LangSmith tracing on/off LIVE —
no backend restart, no admin UI — by flipping a DB-backed flag read once per chat
turn and flipped via a single SQL statement in the Supabase SQL editor.

Both switches feed the SINGLE `tracing_context(enabled=...)` wrapper that 11-05
introduces in the `_traced_turn` seam of chat.py. The composed rule is:

  enabled = langsmith_on AND (not is_user_key)

- Flag OFF  -> enabled=False for EVERYONE -> zero LangSmith runs (live kill-switch,
  beats even owner/demo tracing).
- Flag ON   -> enabled = (not is_user_key) -> BYOK gated off (SEC-01 preserved),
  owner/demo traced exactly as 11-05 leaves it.

Security invariant (non-negotiable): `is_user_key` is resolved LOCALLY and is
INDEPENDENT of the flag read. A failed/missing flag read defaults to True
(default-on) — which is SAFE precisely because the BYOK conjunct always holds:
for a user key, `not is_user_key` is False, so `enabled` is False no matter what
the flag read returns. Default-on can therefore only ever (re)enable tracing for
NON-BYOK turns; it can never leak a BYOK turn.

Purpose: Give the owner an instant, restart-free master kill-switch/enable for
LangSmith without regressing the SEC-01 BYOK confidentiality gate.
Output: migration 034 (app_settings), app_settings_service.py, a one-line refine
of chat.py's 11-05 gate, and two test modules.

Depends on 11-05 (executes first in wave order): this plan refines the exact
`tracing_context(enabled=not is_user_key)` line that 11-05 adds around
`_traced_turn`. An owner-guarded HTTP flip endpoint is explicitly OUT OF SCOPE —
the SQL editor is the flip mechanism for this plan.
</objective>

| Per-turn flag read -> Supabase DB | A flag read on every chat turn is a potential load amplifier and a potential fail-open point if the read errors. |
