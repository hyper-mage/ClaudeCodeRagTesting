<objective>
Close the sole remaining v1.2 milestone blocker: SEC-01 (a). A BYOK user-key chat
turn currently still lands the user's prompt/response in the OWNER's prod LangSmith
project, because Phase 11 gated only the INNER `wrap_openai` client spans
(`get_llm_client(trace=not is_user_key)`), while a PRE-EXISTING outer decorator
`@traceable(name="chat_send_message")` on the endpoint (chat.py:734, present since
Module 1) plus a global `LANGCHAIN_TRACING_V2=true` opens an ungated run for EVERY
turn — capturing `body.content` on entry. Two subagent `@traceable` sites leak the
same way for tool turns (`explorer_service.py:208`, `subagent_service.py:53`).

The gate was applied at the wrong layer. This plan moves the gate to the RUN layer:
drop the endpoint decorator, resolve `is_user_key` BEFORE any traced run opens, and
wrap the turn's work in `langsmith.tracing_context(enabled=not is_user_key)` around a
`@traceable` inner worker. One contextvar suppresses the parent AND — propagating
through `asyncio.to_thread` — both subagent child runs. Owner/demo turns stay fully
traced. A new regression test exercises a full user-key turn including a threaded
tool-call dispatch and asserts ZERO runs — the coverage the old
`test_langsmith_gate.py` (client-wrap only) structurally could not provide.

Purpose: Make a BYOK turn emit zero LangSmith runs; keep owner/demo turns observable.
Output: Fixed `backend/routers/chat.py` + new `backend/tests/test_langsmith_run_gate.py`.

Grounding (verified live against the installed langsmith 0.3.42 in backend/venv):
- `tracing_context` exists with an `enabled` kwarg; `enabled=False` inside a
  `@traceable` fn makes `get_current_run_tree()` return None (no run created);
  `enabled=True` creates a RunTree.
- The `enabled=False` contextvar suppresses NESTED `@traceable` children AND children
  reached via `asyncio.to_thread` (the exact explorer/doc-analysis dispatch pattern).
- `@traceable` wraps an async-generator worker: run opens for the owner turn, none for
  the user-key turn; yields pass straight through to the SSE consumer.
- A `@traceable`-wrapped function carries `__langsmith_traceable__ == True`.
</objective>
