<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → FastAPI (`/api/keys/balance`) | Only the JWT (sub) crosses inbound; no client-supplied threshold or key |
| FastAPI → OpenRouter (`GET /api/v1/key`) | The decrypted sk-or key crosses outbound in the Bearer header; the raw `data` body (label, usage, rate_limit) returns |
| FastAPI → client (response) | ONLY {connected, limit_remaining, is_low} may cross back |
| FastAPI → logs/Sentry | Only scrubbed strings; no key, no Bearer header, no raw provider body |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-01 | Information Disclosure | `GET /api/keys/balance` response | mitigate | Return only `BalanceResponse{connected, limit_remaining, is_low}`; never `resp.json()`/`resp.text`/`data.label`/the key (Task 3 acceptance asserts this) |
| T-14-02 | Information Disclosure | balance-path logging on provider error | mitigate | `logger.warning(scrub_secrets(str(e)))`, fixed generic 502 detail, NO `exc_info` (could capture the Bearer header) — keys.py acceptance greps exc_info==0 |
| T-14-03 | Information Disclosure | decrypted key lifetime | mitigate | `decrypt_key()` result held in a request-local var only; never stored, returned, or logged (reuse Phase 9 control) |
| T-14-04 | Tampering | low-balance threshold | mitigate | `is_low` computed server-side from `get_settings().low_balance_threshold_usd`; threshold never sent to the client, so it cannot be tampered |
| T-14-05 | Spoofing / Elevation | cross-user balance read | mitigate | `Depends(get_user_id)` binds the row lookup to the JWT sub via `.eq("user_id", user_id)`; service-role client scoped by user_id (no cross-user bleed, SEC-04) |
| T-14-06 | Information Disclosure | mid-stream 401 `no_api_key` + 403 `forbidden` SSE branches | mitigate | `_sse_error` wraps detail in `scrub_secrets`; the SSE payload carries only the structured CODE (`no_api_key` / `forbidden`), never the raw provider body or the rejected key |
| T-14-07 | Information Disclosure | `usage` on MessageResponse | accept | `usage` carries cost + token counts only (non-secret); per-user RLS on `messages` already prevents cross-user read |
</threat_model>
