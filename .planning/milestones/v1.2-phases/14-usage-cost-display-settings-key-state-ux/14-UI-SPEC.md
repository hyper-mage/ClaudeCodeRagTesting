# Phase 14 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase scope (from 14-CONTEXT.md):** the *read + render + settings-consolidation* side of cost/usage and key-state UX. The backend capture side already exists (Phase 11: `messages.usage` JSONB, `done` SSE event carries `usage` + `mode:"demo"`, structured error taxonomy `no_api_key`/`rate_limit`/`payment_required`). Six surfaces:

1. **Per-message cost line** — always-visible muted caption under each assistant bubble (`$0.0021 · 1.2k tok`); read from `messages.usage` / the live `done` event, shown as reported (D-01).
2. **Per-thread running total** — muted caption in the active-thread header row (`Σ $0.0142`); summed from persisted `messages.usage.cost` so it survives reload (D-02).
3. **Balance** — surfaced in the Settings OpenRouter section from the new `GET /api/keys/balance`; tolerates null `limit_remaining` for pay-as-you-go (D-04).
4. **Low-balance warning** — amber header status dot + a settings warning line; configurable threshold; no toast/banner (D-03/D-05).
5. **Settings/account page (PREF-01)** — grow the P10 `/settings` stub into three sections: **OpenRouter** (tri-state key status + masked label + balance + disconnect), **Default model**, **Theme**. Absorbs P13's temporary inline default-model + theme mounts (D-06).
6. **Mid-chat key-failure recovery (SC#4)** — extend `ErrorMessageBubble` to render error-type-specific copy + mapped action buttons for 401/402/403 (D-09).

**Explicitly NOT this phase** (do not design): rich model picker w/ favorites/pinning/key-gating (P15), demo-mode banner UI (P15), enabling `demo_fallback_enabled` (P15), profile editing beyond key/model/theme (deferred). The per-thread model selector **stays** in the thread header — it does NOT move to Settings (D-07).

**shadcn gate result: deferred, NOT initialized.** No `components.json` (verified absent at repo + `frontend/`), no `frontend/src/components/ui/` directory, no `tailwind.config.*`. STATE.md records an explicit project decision that `shadcn init` is a **Phase 15** prerequisite (the rich picker's Combobox is the first shadcn surface). Initializing shadcn here would contradict a locked roadmap decision and add an unused dependency surface mid-phase. **Do not run `npx shadcn init` in Phase 14.** Build every new surface as hand-rolled Tailwind consistent with the existing components (`SettingsPage`, `ErrorMessageBubble`, `MessageBubble`, `ConfirmDialog`, `ModelSelector`, `DemoPill`, `DeprecationNotice`).

**Styling system:** Tailwind v4, CSS-first config (`frontend/src/index.css` = `@import "tailwindcss";` + `@plugin "@tailwindcss/typography";`). Theming uses the `class` strategy from Phase 13: `@custom-variant dark (&:where(.dark, .dark *));` with the `<html>.dark` class set pre-mount by the inline script in `index.html`. The four core surfaces resolve via the CSS-variable palette in `index.css` (`--surface-bg`, `--surface-secondary`, `--surface-tertiary`, `--text-primary`, `--text-muted`, `--border`); most components express these as literal `gray-*` utilities with explicit `dark:` variants (the established pattern — match it).

**New theme coverage this phase:** `SettingsPage` is currently **dark-only** (`bg-gray-950` / `bg-gray-900`, no `dark:` variants). Because the Theme toggle now lives *inside* Settings (D-06), the page MUST become theme-aware so light mode reads coherently — apply the core-surface palette below to the page background, the card, and every new control. `ErrorMessageBubble` is likewise dark-only today; add the light-mode error tokens specified in § Color.

## Spacing Scale

Declared values (multiples of 4). Matches the existing 4px-based Tailwind usage already in the codebase (`px-3`=12, `p-4`=16, `p-6`=24, `gap-2`=8, `gap-1`=4, `py-2`=8).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, cost-line dot separator inset (`gap-1`, the `·` middot has `px-1`-equivalent breathing room) |
| sm | 8px | Recovery-button gaps, settings control gaps, dropdown item padding (`gap-2`, `py-2`) |
| md | 12px | Thread-header horizontal padding, error-bubble vertical padding (`px-3`, `py-3`) |
| lg | 16px | Settings page padding, message-list padding, error-bubble horizontal padding (`p-4`, `px-4`) |
| xl | 24px | Settings card inner padding + between-section gaps (`p-6`, `mt-6` between sections) |
| 2xl | 32px | Reserved (not needed this phase) |
| 3xl | 64px | Reserved (not needed this phase) |

**Touch targets:** every interactive control keeps the established **44px** minimum (`min-h-11` / `h-11 w-11`): the Disconnect / Connect / Reconnect / Add-credits buttons, the relocated ThemeToggle (already `h-11 w-11`) and DefaultModelSelector trigger (already `min-h-11`), and each recovery action button in the error bubble. Recovery buttons reuse the existing Retry sizing `px-3 py-2.5 md:py-1.5` (44px on touch, compact on desktop).

Exceptions:
- The **per-message cost line** (`$0.0021 · 1.2k tok`), the **per-thread total** (`Σ $0.0142`), the **balance line**, and the **low-balance warning line** are non-interactive caption text — NOT independent touch targets.
- The **status dot** is the existing `h-2 w-2` (8px) indicator — a non-interactive `role="status"` element, not a tappable control (its `aria-label` carries the state).

## Typography

Matches the existing scale (`text-2xl`/`text-base`/`text-sm`/`text-xs`, weights 400/600/700). Exactly **3 introduced sizes** (12/14/16) at **2 weights** (regular 400 + semibold 600); bold 700 is reserved for the single existing page H1 (`Settings`, `text-2xl font-bold`) and is NOT introduced by this phase.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 | 1.5 — recovery-bubble sentence, settings descriptive copy, masked-label note |
| Label | 14px (`text-sm`) | 600 (`font-semibold`) | 1.4 — button labels (Connect / Disconnect / Reconnect / Add credits), "Connected"/status text, masked label |
| Caption | 12px (`text-xs`) | 400 | 1.4 — per-message cost line, per-thread total, balance line, low-balance warning line, `Connected since`, helper sub-lines |
| Heading | 16px (`text-base`) | 600 (`font-semibold`) | 1.3 — settings section headings (`OpenRouter` / `Default model` / `Theme`) |

Existing (reserved, not introduced): page H1 `Settings` = 24px (`text-2xl`) / 700 / 1.2 — unchanged. ConfirmDialog heading 18px (`text-lg`) / 700 — unchanged (reused as-is for the disconnect confirm).

Notes:
- The per-message cost line and per-thread total are **Caption (12px/400)**, never a heading and never bold — they are quiet metadata, mirroring the `ModelHint` price/context caption and the `DeprecationNotice` line.
- Do not introduce new font sizes. Cost figures, token counts, and balance use Caption; the masked key label keeps the existing `font-mono text-sm`.

## Color

Dark is the existing baseline (verbatim from the codebase). Light is the Phase-13 core-surface palette (`index.css` tokens) — a clean inversion keeping the **same blue accent + red destructive** so the brand reads identically in both themes. Phase 14 adds **amber** as the low-balance/warning semantic and extends light-mode coverage to the Settings page and the error bubble.

### Dark theme (default, existing — do not change)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `gray-950` (#030712) | Page background (chat, settings) |
| Secondary (30%) | `gray-900` (#111827) | Sidebar, header bar, settings card, inputs; `gray-800` (#1f2937) for assistant bubble / hover / chips |
| Accent (10%) | `blue-600` (#2563eb) | See reserved-for list |
| Destructive | `red-600` (#dc2626) | Destructive actions only |

### Light theme (core surfaces — chat, sidebar, settings, composer)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` (#ffffff) | Page background (chat, settings) |
| Secondary (30%) | `gray-50` (#f9fafb) | Sidebar, header bar, settings card; `gray-100` (#f3f4f6) for assistant bubble / hover / chips; inputs `white` + `gray-300` (#d1d5db) border |
| Accent (10%) | `blue-600` (#2563eb) | Same as dark — accent is theme-invariant |
| Destructive | `red-600` (#dc2626) | Same as dark |

**Accent (`blue-600`) reserved for — explicit list, never "all interactive elements":**
- Primary CTAs only: `Connect OpenRouter` (settings, no-key state) and the **primary recovery action** in the error bubble (`Reconnect` for 401/403; `Add credits ⇗` for 402) — styled exactly like the existing `Retry` button (`bg-blue-600 hover:bg-blue-700 text-white`).
- The selected/active model row indicator inside the relocated `DefaultModelSelector` dropdown (`blue-600` check — existing, unchanged).
- Focus rings (`focus-visible:ring-2 focus-visible:ring-blue-500`) — existing convention.
- Inline markdown links in assistant content (`blue-400` dark / `blue-700` light) — existing.
- The model-selector trigger, the ThemeToggle, and the status dot are **NOT** accent.

**Destructive (`red-600`) reserved for:** the `Disconnect` button only (settings) — existing, unchanged. It opens a ConfirmDialog before firing.

**Semantic colors:**

| Role | Dark | Light | Usage |
|------|------|-------|-------|
| Success | `green-500` (#22c55e) dot · `green-400` text | `green-500` dot · `green-600` text (AA on white) | Status dot when connected + balance OK/unknown; "Connected" label |
| Warning / low-balance | `amber-500` (#f59e0b) dot · `amber-300` (#fcd34d) text | `amber-500` dot · `amber-700` (#b45309) text (AA on white/gray-50) | Status dot when connected + balance low; settings low-balance warning line + `AlertTriangle` icon |
| Neutral / not-connected | `gray-500` dot | `gray-400` dot | Status dot when no key connected |
| Error | `red-700` border · `red-950/40` wash · `red-400` icon · `gray-100` text | `red-300` border · `red-50` wash · `red-600` icon · `gray-900` text | Recovery error bubble container (extend `ErrorMessageBubble` with the light tokens) |

**Status-dot tri-state (recolor `IconSidebar` rail dot + `MobileTopBar` dot; D-05):**
- **Connected, balance OK or unknown (null `limit_remaining`)** → `bg-green-500` (unchanged from today).
- **Connected, balance below threshold** → `bg-amber-500`; `aria-label="OpenRouter balance low"`.
- **Not connected** → `bg-gray-500` dark / `bg-gray-400` light (unchanged); `aria-label="OpenRouter not connected"`.
The dot stays `h-2 w-2 rounded-full` and `role="status"`; only the fill color and `aria-label` change. No new shape, no blink/animation.

**Cost-metadata text (muted captions):** per-message cost line, per-thread total, and balance line use the muted-caption token — `gray-400` dark / `gray-600` light. On the assistant bubble (`gray-800` dark / `gray-100` light) `gray-400`/`gray-600` clear WCAG AA. **Do NOT use `gray-500` for the cost line on white** — it is borderline; `gray-600` (#4b5563) is the locked light-mode muted value (Phase 13 contrast guardrail).

**Low-balance warning line (settings):** `amber-700` (light) / `amber-300` (dark) text with a leading `AlertTriangle` (lucide, 14px, `amber-500`). It is colored text + icon inline near the balance — NOT a filled pill, NOT a toast, NOT a blocking banner (D-05). Reuse the DemoPill amber family (`amber-500`/`amber-300`) for token consistency, but no `amber-500/15` background on the warning line (keep it a quiet inline caption).

**Amber is the warning budget, not an accent** — it appears ONLY on the low-balance dot state and the low-balance settings line. It must never style a CTA, a link, or a normal control.

## Copywriting Contract

All copy is locked here (downstream executors must use these exact strings). `${...}` are runtime interpolations of values **as reported by OpenRouter** — never recomputed client-side (ROADMAP SC#1, Phase 11 D-04). The surrounding sentence/format is fixed.

| Element | Copy |
|---------|------|
| **Per-message cost line** (D-01) | `${cost} · ${tokens} tok` — e.g. `$0.0021 · 1.2k tok`. `cost` = `usage.cost` rendered with up to 4 decimals (`$0.0021`); `tokens` = total tokens, formatted `1.2k` for ≥1000 (one decimal, drop trailing `.0`) else the integer (`840`). When `usage.cost` is null/absent (e.g. free model), **omit the cost segment and the `·` separator** — show `${tokens} tok` only. Never invent a cost. |
| **Per-thread running total** (D-02) | `Σ ${total}` — e.g. `Σ $0.0142`. `total` = sum of `usage.cost` across the thread's persisted messages, 4 decimals. **Render only when the summed cost > 0**; omit the element entirely for free-only / empty threads (no `Σ $0.0000` clutter). |
| **Tri-state key status** — demo (D-08) | `Demo mode` (when the `mode:"demo"` signal rides the response/SSE; read it, do not re-derive) |
| **Tri-state key status** — connected | `Your key: connected` (followed by masked label + `Connected since ${date}` + balance line) |
| **Tri-state key status** — no key | `No key — connect to chat` |
| Masked key label | `${status.masked_label}` rendered verbatim in `font-mono text-sm` (existing) |
| Connected-since line | `Connected since ${date}` where `${date}` is `toLocaleDateString('en-US', {month:'short', day:'numeric', year:'numeric'})` — e.g. `Connected since Jun 19, 2026` (existing helper, unchanged) |
| **Balance line** — capped account | `Balance: $${remaining}` — e.g. `Balance: $4.20` (from `limit_remaining`, as reported) |
| **Balance line** — pay-as-you-go (null `limit_remaining`) | `Pay-as-you-go — no limit set` (no warning is computed for uncapped accounts; D-04) |
| Balance line — loading | `Checking balance…` |
| Balance line — fetch failed | `Balance unavailable right now.` (no HTTP code, no provider error — house style) |
| **Low-balance warning line** (D-05) | `⚠ Balance low: $${remaining} — add credits` — e.g. `⚠ Balance low: $0.40 — add credits`. Rendered as the `AlertTriangle` icon + the sentence (do not ship a literal `⚠` glyph if the lucide icon is used; the icon IS the warning mark). Shown only when `limit_remaining` is non-null AND `< LOW_BALANCE_THRESHOLD_USD`. |
| **Recovery — 401 `no_api_key`** (D-09) | Sentence: `Connect your OpenRouter account to keep chatting.` · Buttons: `[Reconnect]` (primary, blue-600) |
| **Recovery — 402 `payment_required`** (D-09) | Sentence: `Your key is out of credit (402).` · Buttons: `[Add credits ⇗]` (primary, blue-600, opens openrouter.ai credits in a new tab `target="_blank" rel="noopener noreferrer"`) + `[Reconnect]` (secondary, neutral) |
| **Recovery — 403** (D-09) | Sentence: `Your key was rejected (403).` · Buttons: `[Reconnect]` (primary, blue-600) + `[Use demo]` (secondary, neutral) — **`[Use demo]` renders only when demo fallback is eligible** (the `mode`/eligibility signal says so); when demo is OFF (the Phase-14 prod default), show `[Reconnect]` alone. |
| Disconnect button | `Disconnect` (existing, unchanged) |
| Connect button (no-key state) | `Connect OpenRouter` (existing, unchanged) |
| Connect helper (no-key state) | `Connect your OpenRouter account to chat with your own key. No key to copy or paste.` (existing, unchanged) |
| Settings section headings | `OpenRouter` · `Default model` · `Theme` (D-06) |
| Default-model heading + helper | `Default model` / `New chats use this model unless you pick a different one for a chat.` (existing `DefaultModelSelector` copy, relocated unchanged) |
| Theme section heading | `Theme` (existing) |
| Primary CTA | `Connect OpenRouter` in the no-key settings state (existing). When connected, there is **no** primary CTA — `Disconnect` is destructive, balance/model/theme are inline controls. |
| Empty state heading | Not applicable as a standalone screen — the **no-key tri-state** (`No key — connect to chat` + the Connect helper + `Connect OpenRouter` CTA) IS the settings "empty" state for the OpenRouter section. |
| Empty state body | `Connect your OpenRouter account to chat with your own key. No key to copy or paste.` (the existing no-key helper) |
| Error state | Balance fetch failure → `Balance unavailable right now.` Mid-chat key failure → the typed recovery sentences above. **Never** interpolate the raw caught error, response body, or `sk-or-…` fragment (SEC-01 + house style); the parenthetical `(401)`/`(402)`/`(403)` is the *structured* error code from the taxonomy, which is allowed. |
| **Destructive confirmation** | **Disconnect** → existing `ConfirmDialog`: heading `Disconnect OpenRouter?`, body `You'll need to reconnect to chat with your own key.`, confirm `Disconnect`, cancel `Cancel`. This is the **only** destructive action in the phase. Changing default model, toggling theme, fetching balance, and tapping a recovery action are all non-destructive — **no confirm dialog** for any of them. |

**Copy rules carried from the codebase (apply to all new strings):**
- Sentence case, no exclamation marks.
- Never interpolate a caught error / HTTP status text / provider name / key fragment into user-facing copy. The locked `Couldn't… / …unavailable right now.` pattern is house style. (Structured taxonomy codes `401/402/403` are the one allowed numeric exception — they come from Phase 11 D-12, not a raw HTTP body.)
- The recovery error bubble is a **persisted thread line** (survives reload), like `DeprecationNotice` — not a toast (D-09). Do NOT also fire a toast for the typed 401/402/403 recovery path; the in-thread bubble is the single surface.
