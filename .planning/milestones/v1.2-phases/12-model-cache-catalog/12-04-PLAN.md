<objective>
Close the single confirmed Phase 12 gap (VERIFICATION truth #5 FAILED / code-review CR-01): the `model_cache.name TEXT NOT NULL` constraint contradicts the no-fallback `model.get("name")` write, so ONE nameless upstream OpenRouter model fails the entire batch upsert. In `refresh_if_stale` that failure is swallowed as "serve stale", leaving a cold/empty cache empty forever (200 []) — violating locked decision D-05 ("never empty") and the phase goal ("current catalog … survives suspend"). In `seed_model_cache.main()` the same failure crashes the deploy. The guarantee currently holds only by luck of present upstream data (0/338 nameless today), not by design.

This plan brings the SQL constraint and the Python write into lockstep with the project's nullable-defensive posture (context_length is nullable; `ModelResponse.name` is `str | None`; `build_model_response` "never raises on a malformed/partial row"), hardens the never-empty fail path (WR-01/WR-02), makes the seed failure visible (IN-01), and locks the fix against silent regression with a constraint-aware test (closing WR-03). Optional cheap wins WR-04 (TTL bound) and WR-05 (no-auth-header invariant test) are folded in.

Purpose: Make "never empty by design" true, not true-by-luck. A nameless upstream model must degrade gracefully (name → model_id) instead of nuking the whole catalog.
Output: corrective migration 031 (applied live to dev), a coalesced + guarded `model_catalog_service`, a non-crashing seed, an extended fixture + regression tests, a bounded TTL, and a no-auth-header invariant test.
