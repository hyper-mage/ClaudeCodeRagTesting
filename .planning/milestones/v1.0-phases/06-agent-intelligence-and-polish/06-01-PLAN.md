    3. Create backend/services/budget_service.py with these functions:

    **count_tokens(text: str) -> int**: Use tiktoken cl100k_base encoding. Return len(encoding.encode(text)). Cache the encoding at module level: `_encoding = tiktoken.get_encoding("cl100k_base")`.

    **count_message_tokens(messages: list[dict]) -> int**: For each message, add 4 tokens overhead (role/separators). Count content tokens via count_tokens. For assistant messages with "tool_calls" key, also count json.dumps(msg["tool_calls"]) tokens. Add 2 tokens at end for reply priming. Return total.

    **fetch_model_context_length(model_id: str, api_key: str) -> int | None**: Use httpx.get to query "https://openrouter.ai/api/v1/models" with Authorization header. Find model by id in response data. Return context_length field. Return None on any exception. Timeout: 10 seconds.

    **class TokenBudget**: Fields: context_length (int), response_reserve (int), safety_margin (float), tool_schema_tokens (int), _system_tokens (int, default 0), _history_tokens (int, default 0), _tool_result_pairs (list of tuples: (pair_index, assistant_tokens, tool_tokens)).

    Properties:
    - available: int(context_length * (1 - safety_margin)) - response_reserve - tool_schema_tokens
    - used: _system_tokens + _history_tokens + sum of all pair tokens
    - is_over() -> bool: used > available
    - remaining: available - used

    Methods:
    - set_system(content: str): _system_tokens = count_tokens(content)
    - set_history(messages: list[dict]): _history_tokens = count_message_tokens(messages) for messages that are NOT tool-related (role in ("user", "assistant") and "tool_calls" not in msg)
    - add_tool_result_pair(assistant_msg: dict, tool_msg: dict): append (len(_tool_result_pairs), count_message_tokens([assistant_msg]), count_message_tokens([tool_msg])) to _tool_result_pairs
    - truncate_oldest_tool_results(messages: list[dict]) -> list[dict]: Find the oldest tool result pair. Remove the corresponding assistant (with tool_calls) message and tool result message from messages list. Remove pair from _tool_result_pairs. Return filtered messages. Per Pitfall 3 from research: always remove the paired assistant+tool messages together to avoid API validation errors.

    **infer_source_scope(user_message: str, has_private_docs: bool) -> str**: Per D-01 and D-02. Keyword heuristic approach (no LLM call). Check for private signals: "my document", "my upload", "uploaded", "my file", "my notes". Check for both signals: "my" + any game/KB reference detected (a rough heuristic). If private signals found and no mixed signals, return "private". If has_private_docs is False, return "default_kb". Otherwise return "both" (D-02: default to both when ambiguous).

    **parse_scope_hint(user_message: str) -> dict**: Parse natural language for scope narrowing. Look for patterns: "only search {X}", "search in {X}", "look in {X}", "just {X}", folder path patterns (containing "/"). If "my uploads" or "my documents" detected, return {"source_hint": "private"}. If "default kb" or "board games only" detected, return {"source_hint": "default_kb"}. If a folder or game name extracted, return {"folder_hint": extracted_name}. Otherwise return {} (no scope hint detected). Per D-09: per-message (stateless) -- no persistence.
