**Fetch logic:**
- `loadFolders`: GET /api/folders with auth header. Transform flat response into nested tree using `buildTree()` (from RESEARCH.md pattern). Separate into two root groups: public folders (Board Games subtree) and private folders (user's folders). Create two virtual root nodes: `{ id: 'root-public', name: 'BOARD GAMES', ... children: publicFolders }` and `{ id: 'root-private', name: 'MY DOCUMENTS', ... children: privateFolders }`. Both virtual roots are always expanded by default.
- `loadContents(folderId)`: GET /api/folders/{folderId}/contents with auth header. For virtual root `root-private`, fetch documents with no folder_id (GET /api/documents for root-level docs).
- `buildBreadcrumbs(folderId)`: Walk up parent chain from selected folder to root. Produces array of `{ id, name }` objects.

**buildTree function** (same file, not exported):
```typescript
function buildTree(folders: FolderRow[]): FolderNode[] {
  const map = new Map<string, FolderNode>()
  const roots: FolderNode[] = []
  for (const f of folders) {
    map.set(f.id, { ...f, children: [] })
  }
  for (const f of folders) {
    const node = map.get(f.id)!
    if (f.parent_id && map.has(f.parent_id)) {
      map.get(f.parent_id)!.children.push(node)
    } else {
      roots.push(node)
    }
  }
  return roots
}
```
