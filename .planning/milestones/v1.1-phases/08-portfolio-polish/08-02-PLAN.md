| Anonymous client → /api/demo/bootstrap | Unauthenticated visitors mint anon JWTs freely; the endpoint must be rate-limited + bounded-work + idempotent. |
| Service-role admin client → auth.users delete | Cascade delete is service-role privileged; cleanup must filter strictly on is_anonymous + age before touching auth.users. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-02 | Denial of Service / Resource leak | /api/demo/bootstrap + cleanup loop | mitigate | Task 2 decorates endpoint with `@limiter.limit("5/minute")` per-IP; cleanup task is bounded to one page (≤100 users) per call per RESEARCH Pitfall 9; idempotency guard skips work when user already has documents. Storage objects deleted BEFORE auth.users delete (RESEARCH Pitfall 2 cascade order). |
| T-08-02-PERM | Elevation of Privilege | /api/demo/bootstrap called by permanent user | mitigate | Task 2 endpoint inspects `user.is_anonymous` via `db.auth.admin.get_user_by_id` and returns `{"seeded": False}` no-op for permanent users — prevents D&D PDF accidentally landing in a permanent user's library. |
| T-08-02-FK | Tampering / Integrity | auth.users delete without ON DELETE CASCADE FK | mitigate | Task 1 `_cascade_delete_user_data` performs explicit child-first deletes in the verified-correct order (document_chunks → documents → folders → messages → threads → storage prefix). Tested in `test_cascade_order`. |
| T-08-02-LEAK | Information Disclosure | Cleanup logs may include user UUIDs | accept | Per-user log lines include user IDs but stay server-side (no Sentry breadcrumbs for backend; Sentry is frontend-only per Phase 7 PII contract). |
| T-08-02-LOOP | Tampering / Integrity | One bad user errors abort the whole purge loop | mitigate | Per-user try/except around `_cascade_delete_user_data` + `delete_user` — logs warning, continues loop. Tested in `test_purge_swallows_per_user_errors`. |
