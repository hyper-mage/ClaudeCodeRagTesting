  <name>Task 1: Author useBodyScrollLock and useSwipeToClose hooks</name>
  <files>frontend/src/hooks/useBodyScrollLock.ts, frontend/src/hooks/useSwipeToClose.ts</files>
  <action>
    Create `frontend/src/hooks/useBodyScrollLock.ts` exporting a named hook `useBodyScrollLock(locked: boolean): void`. Implementation: a single `useEffect` keyed on `locked` that, when `true`, captures `document.body.style.overflow`, sets it to `'hidden'`, and restores the captured value in the cleanup. When `locked` is `false`, the effect is a no-op. This matches CONTEXT.md "Body scroll lock when drawer open" and UI-SPEC.md "Drawer open → `<body>` gets `overflow-hidden` until close".

    Create `frontend/src/hooks/useSwipeToClose.ts` exporting a named hook `useSwipeToClose(onClose: () => void, options?: { thresholdPx?: number; widthPercent?: number }): { onPointerDown, onPointerMove, onPointerUp, onPointerCancel }`. Implementation: use `useRef` for `startX` and `currentDeltaX` (number | null). `onPointerDown(e)` records `startX = e.clientX` and calls `e.currentTarget.setPointerCapture(e.pointerId)` so the gesture survives moves off the element. `onPointerMove(e)` requires `startX != null`; computes `delta = e.clientX - startX`; ignores `delta >= 0` (right-drag); for negative deltas, sets `currentDeltaX = delta`. On `onPointerUp` / `onPointerCancel`, compute drawer panel width from `(e.currentTarget as HTMLElement).offsetWidth`, threshold = `Math.min(options?.thresholdPx ?? 80, width * (options?.widthPercent ?? 0.3))`, and if `Math.abs(currentDeltaX) >= threshold`, call `onClose()`. Always release pointer capture and reset refs in the up/cancel handlers. Do NOT install document-level listeners — the handlers are returned for the caller to spread onto the drawer panel element only (UI-SPEC.md: "gesture scoped to drawer surface only — NOT chat content"). Do NOT animate during drag — drag is detection only; CSS transition handles slide. Per D-decision "Drawer dismiss behavior (full kitchen sink)" and UI-SPEC.md "Hand-rolled pointer events; mirrors CONTEXT.md '50px or 30%' guidance, bumped to 80px".

    Naming conventions per CONVENTIONS.md: camelCase file (`useSwipeToClose.ts`), `use`-prefix hook name, named export (matching `useChat`, `useDocuments` pattern).
  </action>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06.1-01 | Tampering | useSwipeToClose pointer-event capture | accept | Pure client-side UI gesture detection on an authenticated route; worst case is "drawer closes when user didn't intend". No data integrity boundary. |
