# Phase 06.1 — UI Design Contract

> Visual and interaction contract for the mobile-responsive sidebar + hamburger drawer. Generated by gsd-ui-researcher from 06.1-CONTEXT.md + existing frontend tokens. Verified by gsd-ui-checker.

Scope: `AuthenticatedLayout` (App.tsx), `ChatPage`, `DocumentsPage`, `IconSidebar`, `ThreadSidebar`, `ChatContainer` mobile top bar, new `MobileDrawer` + `MobileTopBar` components. Below `md:` (768px) only — desktop ≥768px is **unchanged**.

**Phase-specific dimensional tokens (locked):**

| Token                         | Value          | Tailwind utility           | Notes                                                                                         |
|-------------------------------|----------------|----------------------------|-----------------------------------------------------------------------------------------------|
| Mobile breakpoint             | 768px          | `md:`                      | Below `md:` = mobile drawer mode. ≥ `md:` = desktop unchanged.                                 |
| Drawer width                  | 288px          | `w-72`                     | Inside the 280–320px CONTEXT band; clean Tailwind unit; ≥ 90% of even a 320px viewport.       |
| Hamburger touch target        | 44px × 44px    | `h-11 w-11` (with `p-3`)   | WCAG 2.5.5 / iOS HIG minimum. Icon glyph 24px (`Menu size={24}`).                              |
| Drawer close (X) target       | 44px × 44px    | `h-11 w-11` (with `p-3`)   | Same touch budget. Icon glyph 20px (`X size={20}`).                                            |
| Mobile top bar height         | 48px           | `h-12`                     | Stays above 44px target, matches header heuristic.                                             |
| Drawer slide animation        | 250ms ease-out | `transition-transform duration-250 ease-out` | CSS-only; no JS animation library.                                       |
| Backdrop fade animation       | 200ms ease-out | `transition-opacity duration-200 ease-out` | Slightly faster than slide so backdrop is visible by the time drawer arrives. |
| Swipe-close threshold         | 80px OR 30% of drawer width (whichever smaller) | n/a | Hand-rolled pointer events; mirrors CONTEXT.md "50px or 30%" guidance, bumped to 80px for accidental-swipe resistance. |
| Drawer height (mobile)        | 100dvh         | `h-dvh`                    | Tailwind 4 dynamic viewport — prevents iOS Safari URL-bar gap (CONTEXT.md risk #1).            |
| Drawer z-index                | 50             | `z-50`                     | Above all chat content; backdrop one layer below at `z-40`.                                    |
| Backdrop opacity              | 60%            | `bg-black/60`              | Standard "scrim" treatment over dark theme.                                                    |
| Nav-icons row gap (in drawer) | 8px            | `gap-2`                    | Tight cluster of 3 nav icons (Chat / Documents / Logout).                                      |

**Exceptions:** Drawer width `288px` is a multiple of 4 (`72 × 4`). Top bar height `48px` is `12 × 4`. All other declared values are multiples of 4. No exceptions outside the 4-multiple rule.

## Copywriting Contract

| Element                                | Copy                                       | Source / Rationale                                                                                  |
|----------------------------------------|--------------------------------------------|-----------------------------------------------------------------------------------------------------|
| Primary CTA (in drawer)                | `+ New Chat`                               | Preserved verbatim from existing `ThreadSidebar` — phase is responsive, not copy redesign.          |
| Mobile top-bar title (ChatPage)        | Current thread title (truncate at 1 line); falls back to `New Chat` when title null; falls back to `Chat` when no active thread | Mirrors existing thread fallback `thread.title \|\| 'New Chat'` in ThreadSidebar.   |
| Mobile top-bar title (DocumentsPage)   | `Documents`                                | Matches the route's purpose; no thread context.                                                     |
| Drawer header heading                  | `Menu`                                     | Generic, neutral; the drawer contains both nav and threads, so a generic label avoids over-claiming.|
| Drawer close button (X) — visible      | Icon only (`X` glyph)                      | Standard convention; label conveyed via `aria-label`.                                               |
| Drawer close button — `aria-label`     | `Close menu`                               | Verb + noun; screen reader pronounceable.                                                           |
| Hamburger button — `aria-label`        | `Open menu`                                | Verb + noun; mirrors close.                                                                         |
| Hamburger button — `aria-expanded`     | Bound to `isDrawerOpen` boolean            | A11y contract: announces drawer state.                                                              |
| Hamburger button — `aria-controls`     | `mobile-drawer`                            | Points to drawer element `id="mobile-drawer"`.                                                      |
| Drawer landmark — `aria-label`         | `Navigation and threads`                   | Describes drawer contents to AT users.                                                              |
| Drawer landmark — `role`               | `dialog` with `aria-modal="true"`          | Locks AT focus to drawer contents; matches focus-trap behaviour described under A11y.               |
| Empty state — no threads (in drawer)   | `No conversations yet`                     | Reused from existing implied empty (`ThreadSidebar` renders empty list); explicit now for clarity.  |
| Empty state body — no threads          | `Tap "+ New Chat" to start.`               | Action-directing; references the primary CTA label exactly.                                          |
| Error state — drawer fails to open     | n/a — drawer is local UI state, no async failure path | No network call involved in drawer open/close.                                            |
| Error state — thread list load failure | Existing toast (no change)                 | Reuses `ToastContext` already mounted in `App.tsx`.                                                  |
| Destructive action (in drawer)         | Thread row delete `×` icon                 | Preserved from existing `ThreadSidebar`; no confirm-on-mobile change in this phase.                  |
| Destructive action confirmation        | n/a — none introduced this phase           | Thread delete still uses existing inline `×` without confirm (out of scope).                         |

**Tone:** Imperative, lowercase-after-first-word, no exclamation marks. Match existing app voice ("Send a message to start the conversation.", "+ New Chat").

## Interaction Contract

Locked from CONTEXT.md `<decisions>`; promoted here so executor has a single source of truth.

| Trigger                                          | Behavior                                                                                                  | Implementation note                                                  |
|--------------------------------------------------|-----------------------------------------------------------------------------------------------------------|----------------------------------------------------------------------|
| Tap hamburger (`<md:` only)                      | Drawer slides in from left over 250ms ease-out; backdrop fades in over 200ms ease-out                     | `transform: translateX(-100%) -> 0`                                  |
| Tap backdrop                                     | Drawer slides out; backdrop fades out                                                                     | Click on backdrop element only — not on drawer surface               |
| Tap close (X) inside drawer                      | Same as backdrop close                                                                                    | Shared `closeDrawer()` handler                                       |
| Press `Escape` key while drawer open             | Same as backdrop close                                                                                    | `useEffect` keydown listener mounted only while `isDrawerOpen`       |
| Swipe drawer-left past 80px OR 30% of `w-72` (whichever smaller) | Drawer closes                                                                                | Hand-rolled `pointerdown` / `pointermove` / `pointerup`; gesture scoped to drawer surface only — NOT chat content |
| Tap nav icon (Chat / Documents) inside drawer    | Navigate to route AND close drawer                                                                        | `onClick` calls `navigate(path)` then `closeDrawer()`                |
| Tap Logout inside drawer                         | Sign out AND close drawer                                                                                 | `onClick` calls `signOut()` then `closeDrawer()` (closure order safe — logout redirects to `/login` route which unmounts drawer anyway) |
| Tap thread row inside drawer                     | `onSelectThread(id)` AND close drawer                                                                     | Wrap existing `onSelectThread` callback                              |
| Tap "+ New Chat" inside drawer                   | Create thread AND close drawer                                                                            | Wrap existing `onNewThread` callback                                 |
| Viewport resize crossing `md:` boundary upward   | Drawer state ignored — desktop layout takes over; mobile top bar becomes `md:hidden`                       | CSS-only; no JS resize listener needed                               |
| Drawer open                                      | `<body>` gets `overflow-hidden` until close                                                               | `useEffect` toggling class on `document.body`                        |

**Accessibility contract:**
- Focus moves to the close button (X) on drawer open.
- Focus is trapped inside the drawer while open (only drawer + its children are tab-reachable).
- Focus returns to the hamburger button on drawer close.
- `aria-modal="true"` + `role="dialog"` on drawer container.
- Backdrop is `aria-hidden="true"` (not focusable; just a click target).
- Hamburger button retains `aria-expanded` state synced with `isDrawerOpen`.
