## Decisions Made

- **Per-page drawer state.** Plan-01's CONTEXT.md left "hoisted vs per-page" open. Chose per-page because (a) drawer title differs per route, (b) drawer body content differs per route, (c) per-page state keeps drawer lifecycle scoped to the route's mount/unmount — no cross-route leaks. Hoisting would require passing title + children through `AuthenticatedLayout` with no net simplification.
- **Auto-close only on primary "I'm done choosing" actions.** Thread-select, new-thread, folder-select, and nav buttons close the drawer; delete, rename, create, contextMenu, and external-drop do not. Rationale: those secondary actions open inline editors / context menus that the user may want to complete while the drawer is still open. UI-SPEC.md's Interaction Contract says "tap content row inside drawer closes drawer" — that's primary selection, not destructive/edit ops.
- **Conditional-mount of mobile FolderTree subtree (not the drawer panel itself).** Plan-02's Task 2 critical note flagged the dnd-kit duplicate-id risk: even with `hidden md:flex` on the desktop wrapper, React still mounts both trees in the DOM at `display:none`, and dnd-kit sees duplicate droppable ids regardless of CSS. Gating only the `FolderTree` children on `isDrawerOpen` solves the collision while keeping the drawer panel itself mounted for smooth CSS-transition animations.
- **Task 3 UAT deferred-to-deployed.** Local `.env` is wired to a Supabase project URL that doesn't match the deployed app's prod Supabase project. Running local mobile emulation against a different DB would not be a representative test of what real users will see on `https://boardgame-rag-prod.pages.dev`. Decision (user-approved): mark Task 3 status **DEFERRED-TO-DEPLOYED**, track the full 12-point checklist + 2 a11y sanity checks in `deferred-items.md`, owner=end user, gate phase 06.1 verification on deployed-UAT approval. Phase is NOT marked verified by this summary.
