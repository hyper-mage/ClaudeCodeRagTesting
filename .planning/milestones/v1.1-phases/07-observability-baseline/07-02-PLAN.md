## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| UptimeRobot pinger IPs → /api/health | Unauthenticated public probes. Untrusted source IPs; no user identity. Endpoint MUST NOT leak DB exception details to caller. |
| /api/health → Supabase Postgres | Service-role-keyed read; bypasses RLS by construction. Probe touches `documents` (public-visibility KB seeded in Phase 3). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-08 | Information Disclosure | DB exception text (could leak Postgres connection strings, table names, internal hostnames) returned in HTTP body | mitigate | Failure response is a fixed JSON envelope `{"status": "degraded", "db": "unreachable"}`; the exception goes only to server-side logger via `logger.error(..., exc_info=True)`. RESEARCH ASVS V7 control. |
| T-07-09 | Denial of Service | UptimeRobot 5-min pings (288/day) flooding the rate-limiter and tripping false 429s | mitigate | `/api/health` is left undecorated → slowapi opt-in-per-route mechanism never enforces it (Pitfall 7); test_health_route_has_no_limiter_decorator pins this invariant. |
| T-07-10 | Denial of Service | Slow Postgres causing `/api/health` to block long enough to time out UR (false-positive downtime) | accept | RESEARCH Pitfall 4 + 6 — head-only count is O(microseconds); Phase 4 measured 2s suspend-resume well under UR 30s default timeout; documented mitigation path is bumping UR per-monitor timeout to 45s if false alerts appear. No code-level timeout wrapper added. |
| T-07-11 | Tampering | Future refactor silently changes the DB probe to a no-op (e.g., reverts to `return {"status":"ok"}`) breaking OBS-04 | mitigate | `test_health_actually_calls_supabase_documents_table` asserts on the exact `db.table("documents").select("id", count="exact", head=True).limit(1).execute()` call chain, failing CI if probe is removed/weakened. |
| T-07-12 | Spoofing | Service-role key abused via this endpoint for data exfiltration | accept | Endpoint only invokes a head-only count against a single hardcoded public table; no user input flows into the query; head=True returns no row body. Worst case is the count integer, which `documents` count is already public per Phase 3 default-KB visibility design. |
</threat_model>
