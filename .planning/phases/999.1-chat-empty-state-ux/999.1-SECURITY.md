## Implementation Deviation (T-999.1-03) — documented, not a gap

The PLAN-time mitigation for T-999.1-03 described a literal `if (isStreaming) return` guard inside
`loadMessages`. During Plan 03's human-verify checkpoint (commit `ad43b9a`), that guard was found to
cause a post-stream refetch clobber (the load effect re-fired when streaming stopped) and was
replaced with:
- the send re-entrancy guard reading `isStreamingRef.current` (`useChat.ts:98`), and
- a `[threadId]`-only load effect plus a one-shot `skipNextLoadRef` (`useChat.ts:67-82`).

The **security property the threat asserts is preserved and strengthened**: the load is no longer
destructive to an in-flight send, a concurrent send cannot be starved, and there is no unbounded
loop. The mechanism changed; the DoS-self protection is intact. Verified CLOSED on the property,
with this deviation logged for traceability.

## Accepted Risks Log

The following threats were dispositioned `accept` at plan time. Their accepted-risk entries are
recorded here (required for an `accept` disposition to count as CLOSED).

### AR-999.1-02 — Test fixtures / SSE mock (T-999.1-02)
- **Category:** Information Disclosure
- **Component:** `frontend/src/hooks/useChat.test.tsx`
- **Risk accepted:** Test fixtures could, in principle, embed real credentials.
- **Why acceptable:** Tests mock `../lib/api` (`useChat.test.tsx:8`) and never hit the network; all
  ids are synthetic (`t-new`, `t-existing`, `t-a`, `t-b`); no bearer token, API key, or secret is
  present (grep: 0 matches). No real secret can leak from these fixtures.
- **Verified:** 2026-06-24.

### AR-999.1-05 — Example-prompt chip affordance (T-999.1-05)
- **Category:** Spoofing (UI affordance)
- **Component:** `frontend/src/components/ChatContainer.tsx`
- **Risk accepted:** A suggestion chip could be visually confused with a sent user message.
- **Why acceptable:** Chips render on the gray suggestion surface (`bg-gray-800`,
  `ChatContainer.tsx:56`), visually distinct from the blue (`bg-blue-600`) sent-user bubble; the
  blue fill is absent from ChatContainer (grep: 0 matches). The effect is cosmetic and low risk.
- **Verified:** 2026-06-24.
