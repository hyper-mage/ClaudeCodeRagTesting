## Decisions Made

- **Render backdrop + panel unconditionally, gate via classes.** Mounting both elements regardless of `isOpen` and toggling `opacity-100`/`opacity-0 pointer-events-none` + `translate-x-0`/`-translate-x-full` lets the CSS `transition-opacity duration-200` and `transition-transform duration-250` fire cleanly on every open AND close (vs. conditional mounting which would skip the close animation).
- **`useSwipeToClose` is detection-only.** No DOM mutation during drag (no live-translate); the close animation is entirely owned by the panel's `transition-transform` class. This keeps the gesture logic small (~90 LOC) and avoids the layout-thrash class of bugs that "drag to follow" implementations hit.
- **Added `onPointerCancel` to the swipe API.** Plan called for three handlers; added a fourth so abandoned gestures (interrupt, off-element drag) release pointer capture symmetrically. This is Rule 2 (missing-critical) — without it, pointer capture leaks on cancel events. See Deviations below.
- **`ThreadListContent` is the canonical surface; default `ThreadSidebar` is now just a wrapper.** This puts the empty-state copy in exactly one place and means the desktop rail and mobile drawer (Plan 02) cannot drift apart.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing Critical] Added `onPointerCancel` to `useSwipeToClose` return**
- **Found during:** Task 1 (authoring `useSwipeToClose.ts`)
- **Issue:** Plan's `<action>` specified `{ onPointerDown, onPointerMove, onPointerUp, onPointerCancel }` in the return signature but the prose listed only three handlers ("On `onPointerUp` / `onPointerCancel`, compute drawer panel width...") without explicitly stating that `onPointerCancel` is returned. Without a `pointercancel` handler that releases pointer capture and resets refs, an interrupted gesture (interrupting touch, OS-level cancel, focus loss) would leak pointer capture and leave the swipe state half-set.
- **Fix:** Implemented a shared `finishGesture` helper used by both `onPointerUp` and `onPointerCancel`; both release pointer capture and reset the start/delta refs. Pointer-capture release wrapped in try/catch since some browsers throw when capture was never set.
- **Files modified:** `frontend/src/hooks/useSwipeToClose.ts`
- **Verification:** `npx tsc --noEmit` clean; `npx eslint` clean on the file.
- **Committed in:** `499095b` (Task 1 commit)

**2. [Rule 1 - Bug] Removed bare `flex` from sidebar root classNames when adding `hidden md:flex`**
- **Found during:** Task 3 (editing `IconSidebar.tsx` and `ThreadSidebar.tsx`)
- **Issue:** Pre-existing root classes included `flex flex-col`. The plan specified switching to `hidden md:flex` which only sets `display: flex` at ≥md. Keeping the bare `flex` would have made the sidebar `display: flex` at ALL widths (overriding `hidden`), breaking the mobile hide-out.
- **Fix:** Replaced `flex flex-col` with `flex-col` and let `hidden md:flex` own the `display` token. Plan actually called this out parenthetically ("replace bare `flex` with `md:flex`") so this is on-plan; flagging as deviation only because it required deviation from a literal find/replace.
- **Files modified:** `frontend/src/components/IconSidebar.tsx`, `frontend/src/components/ThreadSidebar.tsx`
- **Verification:** `npm run build` succeeds; `hidden md:flex` grep returns one match per file.
- **Committed in:** `70708f5` (Task 3 commit)
