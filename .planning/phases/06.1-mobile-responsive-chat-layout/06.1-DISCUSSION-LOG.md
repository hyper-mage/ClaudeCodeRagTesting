# 06.1 DISCUSSION LOG

Phase: 06.1 mobile-responsive-chat-layout
Date: 2026-05-14
Discussion mode: discuss (default)

## Origin

Bug surfaced during 06-04 friend-testing of OpenRouter cost guardrail. Mobile users could not interact with chat input because `IconSidebar` (w-14, 56px) + `ThreadSidebar` (w-64, 256px) consume 312px of the 375px viewport. Logged as carry-forward in `06-04-VERIFICATION.md`; inserted as urgent decimal phase 06.1 (after Phase 6) via `gsd-sdk phase.insert`.

## Areas selected

User selected all four offered gray areas (multi-select):
1. Sidebar collapse strategy
2. Breakpoint + scope
3. IconSidebar render inside drawer
4. Drawer dismiss behavior

## Discussion turns

### Turn 1 — Sidebar collapse strategy

**Options presented:**
- Hamburger drawer (slide-in over content) — Recommended
- Collapse-to-icons (push content narrower)
- Hybrid (IconSidebar always visible, only ThreadSidebar drawers)

**Selected:** Hamburger drawer.
**Notes:** Standard mobile pattern; restores full viewport to chat content; hybrid rejected because 56px IconSidebar still meaningful screen surrender on phone.

### Turn 2 — Breakpoint + scope

**Options presented:**
- `md:768px`, ChatPage only — Recommended
- `md:768px`, ChatPage + DocumentsPage
- `lg:1024px`, ChatPage only
- `sm:640px`, ChatPage only

**Selected:** `md:768px`, ChatPage + DocumentsPage.
**Notes:** Larger scope chosen over recommendation; ships full v1.1 mobile polish; both pages share `AuthenticatedLayout` so layout-level fix covers both routes uniformly.

### Turn 3 — IconSidebar inside drawer

**Options presented:**
- Stacked top (nav row + thread list below) — Recommended
- Side-by-side (icon rail + thread list, mirrors desktop)
- Tabs (Threads / Documents / Account)

**Selected:** Stacked top.
**Notes:** Familiar pattern; tap nav icon → navigate + auto-close drawer.

### Turn 4 — Drawer dismiss behavior

**Options presented:**
- Backdrop click + auto-close on select — Recommended
- Backdrop only, no auto-close
- Full kitchen sink (backdrop + swipe + esc + auto-close + close button)

**Selected:** Full kitchen sink.
**Notes:** Portfolio-polish target. Hand-rolled gestures via pointer events to avoid `framer-motion` dependency.

## Claude's discretion (no user input needed)

- Hamburger button location: mobile-only top bar inside ChatContainer/DocumentsPage body
- Drawer state hoisting: in `AuthenticatedLayout` OR per-page (planner picks)
- Body scroll lock: `overflow-hidden` on `<body>` via `useEffect` when drawer open
- iOS Safari viewport: use `h-dvh` (Tailwind 4 dynamic viewport height) instead of `h-screen` on mobile breakpoint to avoid URL-bar gap
- Animation: CSS `transition-transform` for slide; no JS animation library
- Drawer width: ~280–320px (within plan latitude)
