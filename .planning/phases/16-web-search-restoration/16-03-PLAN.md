| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-16-10 | Information Disclosure | key/secret rendered in the failed-state card | accept→mitigated upstream | The tool_result payload contains only `{answer,results}` / `{error: str(e)}` with no key (plan 16-02 keeps the tvly- key in the Authorization header + scrubs); the card adds no new data source. No secret reaches the DOM. |
| T-16-11 | Tampering (XSS) | error text injected into the DOM | accept | The output is rendered in a `<pre>{output}</pre>` as escaped text via React, not `dangerouslySetInnerHTML`; no HTML/script execution surface is introduced by the error branch. |
