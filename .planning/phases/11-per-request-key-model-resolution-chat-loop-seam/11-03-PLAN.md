<objective>
Thread the resolved per-request `api_key` + `model` (and a `trace: bool`) through
`get_llm_client()` and ALL four chat-turn LLM call sites — `stream_chat_completion`
(main), `rerank_service`, `subagent_service`, `explorer_service` — plus the
`retrieval_service.search_documents` → `rerank` hop. Gate `wrap_openai` OFF when the
key is a user key (D-10). Restructure the `stream_chat_completion` stream loop to
drain-and-accumulate the trailing `usage` chunk instead of returning early on
`finish_reason == "tool_calls"` (D-04 / Pitfall 12).

Purpose: This is the SEC-04 plumbing — explicit per-request parameters with NO module
singleton and NO @lru_cache on any key-bearing function (the codebase already builds a
fresh client per call; this is additive params, not a refactor). It also lands the
SEC-01 LangSmith gate at client-construction and the usage-capture restructure that
plan 11-04 consumes. Defaults preserve current owner-key behavior (every new param
falls through to `settings` when unset) so nothing breaks before chat.py wires it.
Output: 5 service files threaded; the wrap gate + usage drain in llm_service.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| per-request key/model state → LLM client construction | The key is now request-scoped; a cached/singleton client would bleed across users. |
| user-key LLM call → LangSmith (wrap_openai) | A wrapped user-key client ships the user's key + full prompt to LangSmith. |
| aux service LLM call → owner key | If only the main completion is threaded, rerank/subagent/explorer silently bill the owner (partial leak defeating D-01). |

| T-11-08 | Information Disclosure | LangSmith wrap on user keys (SEC-01) | mitigate | `trace` param gates `wrap_openai` at construction; user-key calls pass trace=False → wrapper skipped (D-10). test_user_key_client_not_wrapped asserts the bare client.
