<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client request (JWT) → key resolution | `user_id` is JWT-bound (Depends(get_user_id)); the key is read by that user_id only, never from a request-body value. |
| keyless/anon request → owner key | A fail-open fallback would silently bill the owner. The three-branch resolution refuses unless the demo flag is explicitly ON. |
| OpenRouter SDK error → SSE/log | `str(e)` from an OpenRouter error can echo the decrypted `sk-or-…` key out of the trust boundary. |
| decrypted key (in-memory) → trace/log/closure | The plaintext key is a short-lived local; if traced (LangSmith), logged, or captured in a long-lived closure it leaks. |

The fifth owner-key read (budget, Pitfall 1) switched to resolved values. |
