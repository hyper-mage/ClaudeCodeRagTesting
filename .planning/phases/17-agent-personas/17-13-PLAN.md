<objective>
Close Gap 2 from 17-VERIFICATION.md: when a chat is interrupted the backend stamps the assistant row with the sentinel "[Response interrupted]" (chat.py). On reload that row renders as a plain assistant bubble with NO recovery — the user must manually copy/paste and resend the last message. Add a one-click Retry affordance on interrupted assistant turns that re-sends the last user message via the existing send path, mirroring the shipped error-card Retry pattern.

Purpose: bundled UX enhancement (user-requested 2026-07-13). NO PERS requirement — do not map one.
Output: useChat.ts (export the sentinel + extend the retry strip), ChatContainer.tsx (render the interrupted Retry card), ChatContainer.test.tsx + useChat.test.ts (affordance renders + Retry resubmits/strips).

SCOPE NOTE: generic stream errors already render `ErrorMessageBubble` WITH a Retry button, and the model-switch case now renders the typed `model_unavailable` bubble WITH Retry (both shipped). The ONLY uncovered case is the PERSISTED `[Response interrupted]` assistant row (role='assistant', not role='error'), which currently shows no Retry. This plan targets exactly that case — do NOT alter the existing error-bubble paths.
</objective>
