<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| README badges → shields.io | Browsers fetch badge SVGs at render time; shields.io is the trust anchor for badge content. Standard accepted risk for the badge ecosystem. |
| README badges → UptimeRobot public stats | Enabling Public Stats on one monitor exposes a single uptime percentage publicly. Does NOT expose monitor configuration, incident history, or response times unless a public status page is also enabled. Phase 7 rejected a public status page; this plan does not add one. |
| GitHub last-commit badge → repo visibility | Last-commit badge requires Public repo (or a public org). If repo is Private, the badge returns "repo not found" — fallback to a static `version` badge. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-07-SSRF | Information Disclosure | shields.io badge endpoints | accept | shields.io is the standard badge provider for portfolio repos; SSRF mitigation is the host's responsibility. URLs use only documented public endpoints with no user-controlled query params. |
| T-08-07-UR-LEAK | Information Disclosure | UR Public Stats reveals uptime % | accept | A single uptime percentage is non-sensitive metadata. CONTEXT.md D-15 explicitly notes this is the narrower surface vs. a public status page. Phase 7 carve-out preserved. |
| T-08-07-STALE | Tampering / Integrity | Badge SVG could be cached stale by GitHub's image proxy (camo.githubusercontent.com) | accept | GitHub caches badges via Camo; staleness is a few hours at most. Acceptable for a portfolio README — reviewers do not require real-time uptime. |
| T-08-07-VISIBILITY | Information Disclosure | Flipping repo to Public to enable last-commit badge | mitigate | Task 2 explicitly asks the developer to confirm visibility BEFORE flipping; fallback static `version` badge documented if Public is not acceptable. Decision is recorded. |
</threat_model>
