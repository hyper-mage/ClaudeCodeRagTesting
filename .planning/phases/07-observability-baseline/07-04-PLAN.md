<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Developer (in browser) → Sentry SaaS | Auth token used to create releases + upload source maps. Token scoped narrowly: `project:releases` + `org:read` only. |
| CF Pages Build env → @sentry/vite-plugin | Build-time secret exposure to a single dependency that runs only during `npm run build`. Token never reaches Runtime. |
| Deployed CDN → public internet | Bundle + (potentially) source maps available to anyone. Maps must be deleted from dist after upload. |
| End user browser → Sentry SaaS | Public DSN baked into bundle. Acceptable surface; DSN is by design public + rate-limited at Sentry side. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-18 | Information Disclosure | Source-map files publicly served from CDN allowing bundle reverse-engineering | mitigate | `sourcemaps.filesToDeleteAfterUpload` (Plan 07-01) + `curl -I` 404 regression check (final task) |
| T-07-19 | Information Disclosure | `SENTRY_AUTH_TOKEN` accidentally placed in Runtime scope → baked into bundle → exfiltratable from CDN | mitigate | Explicit Build-scope assignment in dashboard (task 2) + SEC-07 leak-grep regression against deployed bundle (final task) |
| T-07-20 | Information Disclosure | JWTs/email/UUID in Sentry event breadcrumbs/request_headers/user fields | mitigate | `beforeSend` + `beforeBreadcrumb` PII scrub from Plan 07-01; final-task verification checks request_headers + breadcrumbs + user fields in the live event |
| T-07-21 | Elevation of Privilege | Compromised `SENTRY_AUTH_TOKEN` (e.g., copy-paste into a chat) used to create fake releases or read other projects | mitigate | Token scoped to `project:releases` + `org:read` only — cannot exfiltrate event data or modify other projects; user instructed to store in password manager; rotate if exposed (task list in SUMMARY) |
| T-07-22 | Denial of Service | Sentry free-tier quota (5k errors/month) exhausted by debug-level event flooding or misconfigured replay sampling | mitigate | `tracesSampleRate: 0.1`, no session replay (Plan 07-01); Sentry SaaS sends quota-warning emails before block (out-of-band) |
</threat_model>
