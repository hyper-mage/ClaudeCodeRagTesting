**Create `backend/models/folder_models.py`** with these Pydantic models (per D-13):

```python
from pydantic import BaseModel

class FolderCreate(BaseModel):
    name: str
    parent_id: str | None = None

class FolderRename(BaseModel):
    name: str

class FolderMove(BaseModel):
    new_parent_id: str | None  # None = move to user root

class FolderResponse(BaseModel):
    id: str
    name: str
    path: str
    parent_id: str | None
    visibility: str
    created_at: str
    updated_at: str
```

**Create `backend/routers/folders.py`** with these endpoints:

1. **GET /api/folders** -- List all folders (user's private + all public). Returns flat list of FolderResponse. Query: `db.table("folders").select("*").or_(f"user_id.eq.{user_id},visibility.eq.public").order("path")`. Frontend builds tree from flat list.

2. **POST /api/folders** -- Create folder. Accept FolderCreate body. If parent_id provided, fetch parent and verify it's not public (403 "Cannot create folders in read-only areas"). Build ltree path: if parent_id, path = `{parent.path}.{sanitize(name)}`; if no parent_id, path = `my_documents_{user_id_prefix}.{sanitize(name)}` where user_id_prefix is first 8 chars of user_id. Set visibility='private', user_id from auth. Return created folder.

3. **GET /api/folders/{folder_id}/contents** -- Get folder contents. Fetch the folder (verify user can see it via user_id match or public). Query subfolders: `db.table("folders").select("*").eq("parent_id", folder_id).or_(f"user_id.eq.{user_id},visibility.eq.public")`. Query documents: `db.table("documents").select("*").eq("folder_id", folder_id).or_(f"user_id.eq.{user_id},visibility.eq.public")`. Return `{"folder": folder_data, "subfolders": [...], "documents": [...]}`.

4. **PATCH /api/folders/{folder_id}** -- Rename folder. Accept FolderRename body. Fetch folder, verify ownership AND visibility='private' (403 if public). Update name. Also update ltree path: replace last segment with sanitize(new_name). Update all descendant paths by replacing old prefix with new prefix (same pattern as move). Return updated folder.

5. **PATCH /api/folders/{folder_id}/move** -- Move folder. Accept FolderMove body. Fetch folder, verify ownership AND private (403 if public). If new_parent_id provided, fetch new parent, verify it's private and owned by user. Compute new_path from new parent. Update folder's parent_id, path. Update all descendant folder paths: query `db.table("folders").select("id, path").eq("user_id", user_id).like("path", f"{old_path}.%")`, then for each descendant update path by replacing old_path prefix with new_path. Also per D-15, no storage path changes needed.

6. **DELETE /api/folders/{folder_id}** -- Delete folder with cascade. Fetch folder, verify ownership AND private (403 if public). Per D-14: (a) Query all documents in this folder and descendants: get all descendant folder IDs via `like("path", f"{folder_path}.%")`, then query documents with folder_id in [folder_id] + descendant_ids. (b) Delete storage files for each document. (c) Delete the folder from DB (ON DELETE CASCADE handles child folders, and document FK cascade handles documents -> chunks). Return 204.

**Helper function** `_sanitize_label(name: str) -> str`: lowercase, replace non-alphanumeric with underscore, collapse multiple underscores, strip leading/trailing underscores. Import `re`.

**Document extensions** -- Add to documents.py:
7. **PATCH /api/documents/{doc_id}/move** -- Move document to folder. Accept `{"folder_id": str | None}`. Verify doc ownership. If folder_id provided, verify folder exists and is private and user owns it. Update doc's folder_id. Return updated doc.
8. **PATCH /api/documents/{doc_id}** -- Rename document. Accept `{"filename": str}`. Verify doc ownership and doc visibility='private'. Update filename. Return updated doc.

**Register router** in `backend/main.py`: add `from routers import folders` to existing import line, add `app.include_router(folders.router)` after documents router.
