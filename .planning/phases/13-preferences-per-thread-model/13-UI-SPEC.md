# Phase 13 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase scope (from 13-CONTEXT.md):** ship the *write + UI* side of model/theme prefs —
(1) per-thread model selector (compact dropdown in the active thread's header),
(2) default-model selector (same minimal component, temporary inline spot),
(3) light/dark theme toggle (server-persisted + localStorage-mirrored, flash-free),
(4) inline deprecation-fallback notice when a pinned model leaves `model_cache`.

**Explicitly NOT this phase** (do not design): rich picker w/ favorites/pinning/key-gating (P15),
settings/account page (P14), usage/cost display (P14). The default-model control is a *temporary*
inline placement that P14 absorbs — keep it small and self-contained.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (hand-rolled Tailwind, no Radix/Base UI) |
| Icon library | lucide-react (`^0.577.0`) |
| Font | system default sans stack (Tailwind v4 default `font-sans`; no custom font configured) |

**shadcn gate result: deferred, NOT initialized.** `components.json`, `tailwind.config.*`, and
`frontend/src/components/ui/` are all absent. STATE.md records an explicit project decision that
`shadcn init` is a **Phase 15** prerequisite (the rich picker's Combobox is the first shadcn surface).
Initializing shadcn here would contradict a locked roadmap decision and add an unused dependency
surface mid-phase. **Do not run `npx shadcn init` in Phase 13.** Build the selector as a hand-rolled
Tailwind dropdown consistent with the existing components (`ThreadSidebar`, `ChatInput`, `ToastContext`).

**Styling system:** Tailwind v4, CSS-first config (`frontend/src/index.css` = `@import "tailwindcss";`
`@plugin "@tailwindcss/typography";`). No JS theme config, no design tokens declared. The app is
**dark-only today** — hardcoded `gray-*`/`blue-*` utilities, **no `dark:` variants anywhere**, no
ThemeProvider. Theming is greenfield wiring (D-01/D-02).

**Theming strategy (Claude's discretion per 13-CONTEXT — DECIDED here):** use a `class` strategy
toggled on `<html>` (`light` / no-class = dark) driven by Tailwind v4's
`@custom-variant dark (&:where(.dark, .dark *));` so existing dark utilities become the **dark**
branch and `light:`-equivalent overrides are added only on the four core surfaces. Equivalent
acceptable alternative: CSS custom properties for the core palette tokens. The contract below
specifies the *resolved color values* for both themes so either mechanism produces the same result.
The localStorage mirror key is `theme` (values `"light"` | `"dark"`), read by a tiny inline script
in `index.html` **before** React mounts to set the root class for flash-free first paint (D-02).

## Spacing Scale

Declared values (must be multiples of 4). Matches the existing 4px-based Tailwind usage
(`p-3`=12, `p-4`=16, `py-2`=8, `gap-2`=8, `gap-4`=16) already in the codebase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, free/paid tag inset (`gap-1`, `px-1`) |
| sm | 8px | Selector row gaps, dropdown item padding (`gap-2`, `py-2`) |
| md | 12px | Header horizontal padding, sidebar padding (`px-3`, `p-3`) |
| lg | 16px | Container padding, message list padding (`p-4`) |
| xl | 24px | Card padding on temporary default-model spot (`p-6`) |
| 2xl | 32px | Reserved (not needed this phase) |
| 3xl | 64px | Reserved (not needed this phase) |

**Touch targets:** interactive controls keep the established **44px** minimum
(`min-h-11` / `h-11 w-11`) — the model-selector trigger and dropdown rows must each be ≥44px tall.

Exceptions: the free/paid tag pill and price/context hint are decorative `text-xs` chips inside a
row and are NOT independent touch targets — they inherit the row's 44px hit area.

## Typography

Matches existing scale (`text-2xl`/`text-base`/`text-sm`/`text-xs`, weights 400/600/700).
Exactly 4 sizes, 2 primary weights (regular 400 + semibold 600; bold 700 reserved for the single
page H1 already in the codebase and not introduced by this phase).

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px (`text-sm`) | 400 | 1.5 |
| Label | 14px (`text-sm`) | 600 (`font-semibold`) | 1.4 — selector trigger model name, section labels |
| Caption | 12px (`text-xs`) | 400 | 1.4 — free/paid tag, price/context hint, deprecation notice, "Using default" sub-line |
| Heading | 16px (`text-base`) | 600 (`font-semibold`) | 1.3 — "Default model" / "Theme" inline section headings |

Notes:
- The **deprecation notice** line is Caption size (12px), NOT a heading — it is a quiet system line,
  not a banner. See Components below.
- Do not introduce new font sizes. The selector trigger uses Label (14/600); dropdown items use
  Body (14/400) for the model name with a Caption (12/400) hint sub-line.

## Color

Dark is the existing baseline (verbatim from `ChatPage`, `ThreadSidebar`, `ChatInput`, `MessageBubble`,
`ToastContext`). Light is the **new** core-surface palette (D-01) — a clean inversion that keeps the
**same blue accent and red destructive** so the brand reads identically in both themes.

### Dark theme (default, existing — do not change)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `gray-950` (#030712) | Page background (chat area, login) |
| Secondary (30%) | `gray-900` (#111827) | Sidebar, header bar, inputs, cards; `gray-800` (#1f2937) for chips / assistant bubble / hover |
| Accent (10%) | `blue-600` (#2563eb) | See reserved-for list |
| Destructive | `red-600` (#dc2626) | Destructive actions only |

### Light theme (new — core surfaces only: chat, sidebar, login, composer)

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `white` (#ffffff) | Page background (chat area, login) |
| Secondary (30%) | `gray-50` (#f9fafb) | Sidebar, header bar, cards; `gray-100` (#f3f4f6) for chips / assistant bubble / hover; inputs `white` + `gray-300` (#d1d5db) border |
| Accent (10%) | `blue-600` (#2563eb) | Same as dark — accent is theme-invariant |
| Destructive | `red-600` (#dc2626) | Same as dark |

**Accent (`blue-600`) reserved for — explicit list, never "all interactive elements":**
- Primary CTAs only: `+ New Chat`, `Send`, `Connect OpenRouter`, `Try the demo` (existing).
- The **selected/active** model row indicator inside the model-selector dropdown (a `blue-600` check
  or left-border on the chosen row).
- Focus rings (`focus:ring-blue-500` / `focus:border-blue-500`) — existing convention.
- Inline markdown links (`blue-400` in dark, `blue-700` in light for contrast).
- The model-selector **trigger is NOT accent** — it is a neutral secondary-surface control
  (`gray-800` dark / `gray-100` light) so it does not compete with the Send CTA.

**Theme toggle is NOT an accent element** — render it as a neutral icon button (lucide `Sun`/`Moon`),
secondary-surface styling, matching the IconSidebar nav buttons.

**Semantic colors (existing, reused — not part of the 10% accent budget):**

| Role | Value | Usage |
|------|-------|-------|
| Success | `green-500` (#22c55e) | Connection dot, "Connected" text (`green-400`) |
| Warning | `amber-600` (#d97706) | Warning toast |
| Error | `red-700` (#b91c1c) | Error toast, error bubble |

**Text colors:**
- Dark: primary `white` / `gray-100`; muted `gray-400` / `gray-500`.
- Light: primary `gray-900` (#111827); muted `gray-600` (#4b5563) / `gray-500`.

**Contrast guardrail:** muted Caption text on its surface must clear WCAG AA (4.5:1 for <18px).
In light mode use `gray-600` (not `gray-400`) for the price/context hint and the "Using default"
sub-line — `gray-400` on white fails AA.

**Deprecation notice colors:** quiet, NOT alarming (it is informational, the thread still works).
Dark: `gray-800` background, `gray-300` text, `gray-700` border. Light: `gray-100` background,
`gray-700` text, `gray-200` border. A muted lucide `Info` icon (`gray-400`/`gray-500`) at 14px.
Do **not** use red/amber — the model simply changed, nothing failed.

## Copywriting Contract

All copy is locked here (downstream executors must use these exact strings). `<default>` and
`<model name>` are runtime interpolations — the surrounding sentence is fixed.

| Element | Copy |
|---------|------|
| Per-thread selector — trigger label | The selected model's display name (`model.name`), or its `id` if name is null |
| Per-thread selector — "using default" trigger sub-state | `Default model` (shown when `threads.model` is null — thread is following the user default; D-05) |
| Per-thread selector — dropdown heading | `Model for this chat` |
| Per-thread selector — "use default" option | `Use my default model` (clears `threads.model` back to null) |
| Default-model selector — section heading | `Default model` |
| Default-model selector — helper sub-line | `New chats use this model unless you pick a different one for a chat.` |
| Free tag (catalog) | `Free` |
| Paid price hint (catalog) | `${price}/M tokens` (rendered from `price_per_mtok_prompt`; show `In ${p} · Out ${c}` only if both present, else the single available figure) |
| Context hint (catalog) | `{context_length / 1000}K context` (omit the line entirely when `context_length` is null) |
| Selector empty/loading state | `Loading models…` |
| Selector error state | `Couldn't load models. Tap to retry.` |
| Theme toggle — accessible label (to light) | `Switch to light theme` |
| Theme toggle — accessible label (to dark) | `Switch to dark theme` |
| Theme section heading (temp spot) | `Theme` |
| **Deprecation notice (SC#4)** | `Model "<model name>" is no longer available — using <default> instead.` |
| Primary CTA | (none new this phase — selectors are inline controls, not CTAs; existing `+ New Chat` / `Send` unchanged) |
| Empty state heading | not applicable — Phase 13 adds controls to populated surfaces; the chat empty state is owned by 999.1 and unchanged |
| Empty state body | not applicable (see above) |
| Error state | Selector load failure: `Couldn't load models. Tap to retry.` — no provider name, no HTTP code (matches the codebase's locked generic-error convention) |
| Destructive confirmation | **None.** Picking a per-thread model and clearing back to default are both non-destructive and instantly reversible — **no confirm dialog.** Changing theme and changing default model are likewise non-destructive. Do not add a ConfirmDialog for any Phase 13 action. |

**Copy rules carried from the codebase (apply to all new strings):**
- Sentence case, no exclamation marks.
- Never interpolate a caught error / HTTP status / provider name into user-facing copy
  (the `Couldn't…` pattern is the locked house style — see LoginPage / ChatPage toasts).
- The deprecation notice is a **persisted thread line** (survives reload), not a toast (D-06).

## Components (this phase)

Hand-rolled, no shadcn. All must support BOTH themes via the class strategy above.

| Component | Placement | Behaviour |
|-----------|-----------|-----------|
| `ModelSelector` (reusable) | core control | Trigger button (Label type, secondary surface, lucide `ChevronDown`) → popover list of `GET /api/models` rows. Each row: model name (Body) + Caption hint (`Free` tag OR price + context). Selected row marked with `blue-600` check/left-border. Keyboard: `Enter`/`Space` opens, `Esc` closes, arrow-key navigation, focus trap while open, focus returns to trigger on close. Closes on outside-click and on select. ≥44px rows. `aria-haspopup="listbox"`, `aria-expanded`, `role="listbox"`/`role="option"`. |
| Per-thread selector | **Active thread header** (a NEW header row in `ChatContainer`, above the message scroll area — none exists today; add `h-12`, `gray-900`/`gray-50`, `border-b border-gray-800`/`gray-200`, `px-3`, matching `MobileTopBar`). On mobile, place it in/under `MobileTopBar`. | Uses `ModelSelector`. On select → `PATCH /api/threads/{id}` `{model}`; on "Use my default model" → PATCH with `{model: null}`. Trigger shows `Default model` sub-state when `threads.model` is null. Only renders when a thread is active (hidden on cold-start empty state). |
| Default-model selector | **Temporary inline spot** — the desktop `ThreadSidebar` footer (below the thread list) and the `MobileDrawer` (below `IconNavRow`). Keep it visually quiet; P14 moves it to the settings page. | Uses `ModelSelector` with the `Default model` heading + helper sub-line. On select → `PUT /api/preferences` `{default_model}`. |
| `ThemeToggle` | Same temporary spots as the default-model control (sidebar footer + mobile drawer). Optionally also the IconSidebar rail bottom. | Neutral icon button (`Sun`/`Moon`, 20px), toggles root class + writes `localStorage.theme` synchronously, then `PUT /api/preferences` `{theme}` (fire-and-forget; localStorage is the source of truth for paint). `aria-pressed` reflects current theme. |
| `DeprecationNotice` | Inline in the message stream at send time (SC#4 / D-06) | A persisted non-AI system line styled per the Deprecation-notice colors above: full-width centered row, muted `Info` icon + Caption text, NOT a message bubble, NOT a toast, NOT red. Reuses the visual language of a system divider. Persists in thread, visible on reload, must not crash the thread. |

**Consistency requirements:**
- Selector trigger matches `ChatInput`/`ThreadSidebar` button radii (`rounded`), border, and focus
  treatment (`focus:outline-none focus:ring-2 focus:ring-blue-500`).
- The new chat-header row must not break the existing `flex-1 flex flex-col h-full` layout of
  `ChatContainer` — it is a `shrink-0` row above the scroll area.
- Light mode must produce a *coherent* surface on chat/sidebar/login/composer — no orphaned
  `gray-950` panels bleeding through (D-01). Secondary UI (Documents page, tool cards) may stay dark
  this phase; the toggle is acceptable as long as the four core surfaces read correctly.
