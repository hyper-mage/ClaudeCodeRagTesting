## ToolCallCard Specification

### Structure

```
+-----------------------------------------------+
| [icon] Tool Name   args="value"    [chevron]  |
+-----------------------------------------------+
| (collapsed by default)                         |
| Output content in monospace...                 |
| $ path/to/file.md                              |
|   matched line content                         |
+-----------------------------------------------+
```

### Layout Details

- Full-width within the message bubble (max-w matches parent)
- Rounded corners: rounded-md (6px)
- Border: 1px solid gray-700
- Margin between cards: 8px (gap-2 on the card container)
- Cards appear above the assistant message text content

### Header Row

- Flex row, items-center, justify-between
- Left side: icon (16x16) + 4px gap + tool name (semibold, 12px) + 8px gap + arguments (mono, 12px, gray-400, truncated with ellipsis at 200px max-width)
- Right side: spinner (when status=running, 14x14 border spinner) OR check icon (when status=complete, 14x14 lucide Check, gray-500) + chevron (ChevronDown/ChevronUp, 14x14, gray-500)
- Padding: 8px horizontal, 6px vertical
- Cursor: pointer (entire header is clickable to toggle)

### Output Area (Expanded)

- Border-top: 1px solid gray-700
- Padding: 16px horizontal, 12px vertical
- Font: monospace, 12px, gray-300
- Max-height: 300px with overflow-y: auto
- White-space: pre-wrap (preserve formatting of ls/tree/grep output)
- Empty state (tool_start received, no result yet): show nothing, just the spinner in header

## Interaction States

### ToolCallCard States

| State | Header Visual | Output Area | Trigger |
|-------|---------------|-------------|---------|
| Running + Collapsed | Icon + name + args + spinner + chevron-down. Background: gray-800/50 | Hidden | SSE `tool_start` event received |
| Running + Expanded | Same as above but chevron-up | Visible, empty (no content yet) | User clicks header while running |
| Complete + Collapsed | Icon + name + args + check + chevron-down. Background: gray-800/50 | Hidden | SSE `tool_result` received (default) |
| Complete + Expanded | Same as above but chevron-up. Background: gray-800 | Visible, shows tool output | User clicks header after completion |

### SSE Event Flow

1. Backend yields `tool_start` event: `{"tool_event": true, "type": "tool_start", "tool": "kb_ls", "args_preview": "path=\"Board Games/\"", "call_id": "abc123"}`
2. Frontend creates ToolCallCard in Running+Collapsed state
3. Backend executes tool, yields `tool_result` event: `{"tool_event": true, "type": "tool_result", "tool": "kb_ls", "call_id": "abc123", "output": "Catan/\nTicket to Ride/\n..."}`
4. Frontend updates card to Complete+Collapsed state, stores output
5. User can click to expand/collapse at any time

### ToolEvent Interface (updated)

```typescript
interface ToolEvent {
  tool: string
  args_preview: string
  output?: string
  call_id?: string
  subagent?: boolean
  status: 'running' | 'complete'
}
```

### Tool Card Argument Display

Arguments are displayed as a key=value string in the card header. Examples:
- kb_ls: `path="Board Games/"`
- kb_tree: `depth=2`
- kb_read: `path="Board Games/Catan/rules.md" lines="1-50"`
- kb_grep: `pattern="worker placement" mode="keyword"`
- kb_glob: `pattern="Board Games/*/rules.md"`

These strings come from the backend `args_preview` field (existing pattern, extended to new tools).
